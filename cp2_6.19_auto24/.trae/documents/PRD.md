# 会议反馈收集与可视化系统 PRD

## 1. 项目概述

### 1.1 项目背景
团队会议结束后，反馈意见分散在各个渠道，难以有效汇总和跟踪改进。本系统旨在提供一个集中式的会议反馈收集、可视化和跟踪平台，帮助团队高效收集反馈、分析数据、跟踪改进措施。

### 1.2 目标用户
- 会议组织者/主持人
- 会议参与者
- 团队管理者

### 1.3 核心价值
- 快速收集结构化会议反馈
- 可视化展示反馈数据，洞察会议质量趋势
- 跟踪反馈处理状态，确保改进落地

---

## 2. 功能需求

### 2.1 反馈表单模块
**用户故事**：作为会议参与者，我希望能够快速提交结构化的会议反馈，以便组织者了解我的意见。

**功能描述**：
- 反馈人角色选择：主持人、参会者、旁听（下拉选择）
- 满意度星级评分：1-5星（点击选择，悬停预览）
- 关键收获文本输入（多行，字符计数提示）
- 待改进点建议（多行，字符计数提示）
- 表单字段焦点放大动画
- 提交按钮按压反馈效果
- 表单校验：必填字段校验

### 2.2 反馈卡片模块
**用户故事**：作为会议组织者，我希望能够查看所有反馈并展开查看详情，以便全面了解反馈内容。

**功能描述**：
- 卡片按提交时间倒序排列
- 缩略状态显示：评分星标 + 关键收获前20字摘要
- 展开/收起功能，平滑过渡动画（300ms，缓动函数）
- 展开显示完整的关键收获和待改进点
- 已处理反馈标记：绿色边框 + 顶部对勾图标
- 已处理反馈自动置底（向上滑动归位动画）

### 2.3 综合仪表盘模块
**用户故事**：作为团队管理者，我希望能够查看会议反馈的综合统计，以便评估会议整体质量。

**功能描述**：
- 平均分折线图：横轴为反馈序号，纵轴为分数（0-5）
- 满意度分布环形图：1-5星各占比例
- 关键词云：从关键收获文本提取高频词，词频越高字号越大
- 响应式布局：桌面端左右并排，移动端上下堆叠
- 图表渲染时间 ≤ 500ms（50条数据）

### 2.4 反馈处理模块
**用户故事**：作为会议创建者，我希望能够标记反馈为已处理并回复改进措施，以便跟踪问题解决。

**功能描述**：
- 标记反馈为已处理
- 输入改进措施回复
- 处理后卡片视觉变化：绿色边框 + 对勾图标
- 已处理反馈自动置底，带动画效果

### 2.5 历史会议概览模块
**用户故事**：作为用户，我希望能够浏览历史会议记录，以便查看过往会议的反馈情况。

**功能描述**：
- 按月份分组显示会议卡片
- 卡片显示：会议名称、创建时间、参与人数、平均分
- 卡片背景色根据平均分渐变（高分绿、低分红）
- 点击跳转至会议详情页

## 3. 界面设计规范

### 3.1 色彩系统
- 主色调：沉稳蓝白灰
  - 主蓝：#2563eb
  - 深蓝：#1e40af
  - 浅灰：#f3f4f6
  - 中灰：#9ca3af
  - 深灰：#374151
- 强调色：
  - 橙色（按钮/高亮）：#f97316
  - 绿色（已处理/高分）：#10b981
  - 红色（低分）：#ef4444

### 3.2 视觉设计
- 卡片：圆角 12px，阴影 elevation-2，悬停时 elevation-4 + 轻微放大
- 背景：浅灰色磨砂质感（#f8fafc + 细微噪点）
- 导航栏：桌面端左侧固定，移动端汉堡菜单
- 页面过渡：淡入淡出 300ms

### 3.3 交互动效
- 可交互元素悬停：放大 1.02 倍，阴影加深
- 表单字段聚焦：scale(1.02)，边框高亮
- 按钮按压：scale(0.97)
- 卡片展开/收起：height 过渡 300ms，ease-in-out
- 列表项位置变化：translateY 过渡 300ms

---

## 4. 性能要求

| 指标 | 要求 |
|------|------|
| 仪表盘图表渲染时间 | ≤ 500ms（50条反馈数据） |
| 列表滚动帧率 | ≥ 60fps |
| 展开收起动画帧率 | ≥ 30fps |
| 页面切换过渡 | 流畅无卡顿 |

---

## 5. 技术约束

- 前端框架：React 18 + TypeScript
- 构建工具：Vite
- 状态管理：Zustand
- 图表库：Recharts
- 数据存储：localStorage（模拟后端）
- ID 生成：uuid

---

## 6. 非功能需求

### 6.1 响应式设计
- 桌面端（≥ 1024px）：左侧导航栏 + 主内容区
- 平板（768px - 1023px）：导航栏可折叠，图表上下堆叠
- 移动端（< 768px）：汉堡菜单，单列布局

### 6.2 数据持久化
- 所有会议和反馈数据存储在 localStorage
- 页面刷新后数据不丢失

---

## 7. 验收标准

1. ✅ 用户可创建会议并提交反馈
2. ✅ 表单字段有正确的动画效果和计数提示
3. ✅ 反馈卡片支持展开/收起，动画流畅
4. ✅ 仪表盘正确显示三种图表，响应式布局
5. ✅ 可标记反馈为已处理并回复，视觉状态正确
6. ✅ 历史会议按月份分组，卡片颜色随分数渐变
7. ✅ 所有数据持久化到 localStorage
8. ✅ 性能指标达标

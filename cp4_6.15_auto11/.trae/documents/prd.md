## 1. 产品概述
轻量级多人费用分摊计算器，解决小型团队或家庭在聚餐、旅行、合租等场景下费用分摊逻辑复杂、零钱结算繁琐、容易产生分歧的问题。

- 核心价值：通过自动化算法简化多人费用分摊和债务结算，最小化转账次数
- 目标用户：3-10人的小型团队、家庭、室友、旅行伙伴等
- 产品定位：清爽、高效、无社交负担的纯粹费用分摊工具

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册，本地存储 | 创建/编辑费用、管理参与人、生成分摊方案、查看历史记录 |

### 2.2 功能模块
1. **费用管理模块**：费用录入、分摊规则设置、费用列表展示、费用详情查看
2. **参与人管理模块**：动态添加/删除参与人、设置昵称和emoji头像
3. **结算引擎模块**：最小化转账次数算法、结算建议生成、手动调整金额
4. **历史记录模块**：已结算费用归档、搜索筛选、滑动删除

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|---------|----------|----------|
| 首页 | 费用输入表单 | 输入金额、选择付款人、填写描述、设置分摊方式（均分/按比例/指定人） |
| 首页 | 参与人列表 | 动态增删参与人、设置昵称和头像、权重拖拽调整 |
| 首页 | 费用列表 | 展示所有未结算费用、点击查看详情、滑动删除 |
| 结算页面 | 结算建议卡片 | 展示最优转账方案、SVG箭头动画、点击调整金额或忽略 |
| 历史页面 | 历史记录 | 按日期归档已结算记录、搜索筛选、滑动删除 |
| 详情弹窗 | 分摊明细 | 展示每人应付/已付/差额、行色差区分、高亮当前用户 |

## 3. 核心流程

用户添加参与人 → 录入多笔费用并设置分摊规则 → 系统实时计算每人债务 → 点击结算生成最优转账方案 → 用户确认转账后标记为已结算 → 记录归档到历史

    E --> F["确认转账"]

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：冷灰色基底（#f8fafc, #f1f5f9）搭配薄荷绿（#34d399）与淡珊瑚色（#f87171）作为强调色
- **按钮样式**：圆角8px，按下缩放0.95，涟漪波纹效果
- **字体**：现代无衬线字体，标题600字重，正文400字重
- **布局风格**：卡片式布局，微妙投影（0 2px 8px rgba(0,0,0,0.08)），圆角12px
- **emoji头像**：内置12个常用emoji供选择（👤, 👨, 👩, 🧑, 👴, 👵, 👶, 👱, 👲, 👳, 🧔, 👸）
- **动画**：CSS transition 200ms cubic-bezier(0.4, 0, 0.2, 1)，SVG路径动画

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|---------|----------|--------|
| 首页 | 费用输入卡片 | 卡片投影、聚焦底部高亮过渡、分摊方式切换标签、权重滑块 |
| 首页 | 参与人卡片 | 头像选择器、昵称输入、平滑添加/删除动画 |
| 首页 | 费用列表 | 左滑删除手势、卡片消失缩小动画、色差行区分 |
| 结算页面 | 转账卡片 | SVG箭头路径动画、发送方/接收方头像、金额可编辑 |
| 历史页面 | 记录列表 | 日期分组、搜索框、筛选标签、滑动删除 |
| 详情弹窗 | 分摊明细表 | 淡入淡出、行色差、当前用户高亮、差额正负颜色区分 |

### 4.3 响应式设计
- 桌面端（≥1024px）：左侧可折叠侧边栏导航，主内容区三栏布局
- 平板端（768px-1023px）：顶部导航栏，双栏内容布局
- 手机端（<768px）：底部固定导航栏，单列内容布局，触摸优化（44px最小点击区域）

### 4.4 性能要求
- 所有交互响应时间 ≤ 100ms
- 重算动画保持 60FPS
- 使用 CSS transform 和 opacity 实现动画，避免触发重排
- 防抖处理频繁触发的操作（如拖拽权重）

## 1. 产品概述

乐队巡演协作管理平台，帮助小型独立乐队在线创建分享巡演路线、管理场地排期、统一管理歌单并实现乐队成员间的高效协作。解决乐队在组织小型巡演时沟通成本高、路线规划繁琐、歌单信息分散在多个平台难以统一管理的痛点。

- 核心用户：小型独立乐队的队长（管理员）和乐队成员
- 核心价值：巡演全流程数字化管理 + 成员实时协作 + 歌单统一调度

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 乐队队长（管理员） | 邮箱注册 | 创建/编辑巡演路线、添加成员、管理歌单、查看所有排期 |
| 乐队成员 | 邮箱注册 + 队长邀请加入 | 编辑个人角色/状态、更新场次参与状态、查看仪表盘、浏览歌单 |

### 2.2 功能模块

1. **巡演路线规划模块**：地图城市选择、路线绘制、场次编辑、时间线展示
2. **成员管理与协作模块**：成员邀请、角色管理、排期状态、WebSocket实时同步
3. **歌单管理模块**：歌单创建、歌曲增删改、拖拽排序、场次绑定
4. **成员仪表盘模块**：三列卡片分类（即将到来/待确认/历史）、场次详情、参与反馈

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 巡演路线编辑页 | 地图交互区 | 在地图上按顺序选择3-10个城市，自动绘制折线，路线颜色可自定义 |
| 巡演路线编辑页 | 城市节点弹窗 | 点击城市节点弹出编辑窗口，可修改场地名称、演出日期、开始时间、上传海报 |
| 巡演路线编辑页 | 场次时间线 | 右侧垂直布局时间线，圆角矩形卡片展示场次，悬停展开显示完整信息和海报缩略图（0.3s动画） |
| 巡演路线编辑页 | 成员排期表 | 每场下展示成员参与状态（参加/不参加/待定），实时同步 |
| 巡演路线编辑页 | 歌单绑定区 | 为每场绑定主歌单和安可歌单 |
| 歌单管理页 | 歌单列表 | 创建主歌单/安可歌单，表格展示歌曲序号、名称、时长、调式 |
| 歌单管理页 | 歌曲编辑 | 点击行展开编辑备注，支持添加歌曲（名称、时长mm:ss、调式、备注） |
| 歌单管理页 | 拖拽排序 | 拖拽调整顺序，半透明占位条，0.2s重排动画 |
| 成员仪表盘 | 三列卡片 | 即将到来（蓝色渐变）、待确认（橙色渐变）、历史（灰色渐变），毛玻璃背景 |
| 场次详情页 | 信息展示 | 地图缩略图、日期场地、绑定歌单 |
| 场次详情页 | 参与反馈 | 成员直接选择参加/不参加/待定，实时同步 |
| 成员设置页 | 个人信息 | 编辑角色（主唱/吉他/贝斯/鼓/键盘）和状态（活跃/休息中） |

## 3. 核心流程

### 3.1 队长创建巡演流程

队长登录后进入巡演创建页，在地图上依次点击3-10个城市，系统自动连线；逐个点击城市节点编辑场地、日期、时间并上传海报；完成后系统生成时间线展示所有场次；队长添加成员并分配角色；创建歌单并绑定到各场次。

```mermaid
flowchart LR
    A["队长登录"] --> B["创建新巡演"]
    B --> C["地图选城市(3-10个)"]
    C --> D["编辑城市场次信息"]
    D --> E["生成场次时间线"]
    E --> F["添加乐队成员"]
    F --> G["创建主/安可歌单"]
    G --> H["绑定歌单到各场次"]
    H --> I["发布巡演路线"]
```

### 3.2 成员协作流程

成员通过邀请链接注册并加入乐队；登录后仪表盘展示相关场次；点击场次卡片进入详情页，反馈参与状态；状态变更通过WebSocket实时同步到队长端和其他成员端。

```mermaid
flowchart LR
    A["成员注册/登录"] --> B["查看仪表盘"]
    B --> C["点击场次卡片"]
    C --> D["进入场次详情页"]
    D --> E["反馈参与状态(参加/不参加/待定)"]
    E --> F["WebSocket广播状态变更"]
    F --> G["所有设备实时同步"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深紫色（#1a1a2e）与亮紫色（#9b59b6）渐变背景
- **强调色**：金色（#f1c40f）用于按钮、高亮和重要标识
- **主题**：深色主题，所有卡片和按钮采用圆角设计（圆角半径 8-16px）
- **按钮效果**：悬停时放大1.05倍 + 金色光晕效果，transition 0.2s
- **页面切换**：内容以0.4秒淡入效果切换
- **毛玻璃效果**：仪表盘卡片、时间线卡片使用 backdrop-filter: blur(10px)

### 4.2 页面布局

| 页面名称 | 区域布局 | UI元素 |
|----------|----------|--------|
| 巡演路线编辑页 | 左侧65%地图 + 右侧35%详情面板（移动端上下各50%） | Leaflet地图、城市节点Marker、可编辑路线、时间线卡片、排期表格 |
| 歌单管理页 | 顶部导航 + 主体双栏（歌单列表 + 歌曲表格） | 圆角卡片、表格行展开、拖拽占位条、重排动画 |
| 仪表盘 | 顶部导航 + 三列卡片网格（<768px改为单列） | 蓝色/橙色/灰色渐变卡片、毛玻璃背景、悬停放大 |
| 场次详情页 | 顶部返回 + 地图缩略图 + 信息分区卡片 | 地图预览、状态切换按钮、歌单列表 |

### 4.3 响应式设计

- **桌面端（≥768px）**：巡演编辑页采用左右布局（地图65%/面板35%），仪表盘三列网格
- **移动端（<768px）**：巡演编辑页上下布局（地图50%/面板50%），仪表盘单列垂直排列
- **触摸优化**：拖拽操作支持触摸事件，点击区域最小44×44px

### 4.4 动画与动效

- 时间线卡片悬停展开：0.3s ease-out，向右扩展显示海报
- 歌单拖拽排序：半透明占位条 + 0.2s重排动画
- 按钮悬停：scale(1.05) + 金色 box-shadow 光晕
- 页面切换：opacity 0 → 1，0.4s ease-in
- 地图城市节点：弹跳出现动画，点击脉冲反馈

# 风味轮盘 (Flavor Wheel) 产品需求文档

## 1. 项目概述

### 1.1 项目背景
风味轮盘是一款面向美食爱好者、专业品鉴师和餐饮从业者的数字化品鉴记录工具。通过直观的视觉化交互，用户可以为菜品的六个核心风味维度（甜、酸、苦、辣、咸、鲜）进行量化评分，并生成专业的风味轮廓图。

### 1.2 核心价值
- 将抽象的味觉体验转化为可视化的数字记录
- 提供专业级的品鉴记录与分享功能
- 建立个人美食品鉴数据库

### 1.3 目标用户
- 美食爱好者和美食博主
- 专业厨师和菜品研发人员
- 餐饮行业品鉴师

### 1.4 设计理念
科技感暗色调设计，突出数据可视化的专业感，同时保持操作的流畅性。

## 2. 产品功能

### 2.1 核心功能模块

#### 2.1.1 风味轮盘交互
- 中央Canvas绘制的风味轮盘（直径320px）
- 六个风味维度：甜、酸、苦、辣、咸、鲜
- 每个维度0-9刻度评分
- 六根可拖拽指针，沿弧线移动
- 拖拽时实时更新雷达图
- 鼠标悬停扇区放大1.05倍并显示标签

#### 2.1.2 极坐标雷达图
- 与轮盘同宽320px
- 六边形网格线
- 数据点连线白色实线（2px）
- 半透明白色填充（0.3）

#### 2.1.3 品鉴记录表单
- 菜品名称输入框（实时校验非空）
- 品鉴日期选择器（Gpickr风格）
- 多行文字描述（最多200字，字数实时显示）
- 照片上传（支持jpg/png，≤5MB，自动裁剪为正方形200px）
- 所有输入实时表单校验

#### 2.1.4 历史记录面板
- 右侧固定面板（宽300px）
- 卡片式记录列表（每张高100px）
- 左边缘4px色条标识主味
- 点击卡片弹出详情弹窗
- 支持按日期/维度排序

#### 2.1.5 分享名片卡生成
- 384x216px PNG图片
- 风味轮盘缩略图
- 菜品名和总评语
- 生成时间和二维码占位
- 支持下载或复制到剪贴板

### 2.2 交互设计

#### 2.2.1 动画效果
- 轮盘旋转：cubic-bezier(0.23, 1, 0.32, 1)，0.4s
- 面板滑入：ease-out，0.3s
- 弹窗渐入：ease，0.3s
- 按钮点击缩放：0.95
- 状态变化平滑过渡

#### 2.2.2 响应式布局
- 桌面端（≥1024px）：三栏布局，历史面板右侧固定
- 平板端（768px-1023px）：自适应布局
- 移动端（≤767px）：历史面板折叠为底部抽屉

#### 2.2.3 性能要求
- 轮盘拖动和雷达图更新：60fps
- 轮盘旋转动画：≥50fps
- 页面加载时间：≤2s

### 2.3 视觉设计

#### 2.3.1 色彩系统
- 主色调：#1A1A2E（深背景）、#2D2D44（次背景）
- 点缀色：
  - 甜：#FF9F43
  - 酸：#EE5A24
  - 苦：#6C5CE7
  - 辣：#FD79A8
  - 咸：#00CEC9
  - 鲜：#FDCB6E
- 边框/分割线：#4A4A6A
- 文字：白色

#### 2.3.2 视觉效果
- 径向渐变背景
- 半透明扇区填充（透明度0.6）
- 按钮悬停发光效果
- 卡片玻璃拟态风格
- 滚动条自定义

## 3. 技术架构

### 3.1 技术栈
- 框架：React 18 + TypeScript
- 构建工具：Vite
- 状态管理：Zustand
- 图像处理：html-to-image + file-saver
- 唯一标识：uuid

### 3.2 项目结构
```
src/
├── main.tsx              # 入口文件
├── stores/
│   └── appStore.ts     # Zustand全局状态
├── hooks/
│   └── useWheelEngine.ts # 轮盘交互逻辑
├── components/
│   ├── WheelCanvas.tsx  # Canvas轮盘组件
│   ├── FlavorRadar.tsx # 雷达图组件
│   ├── TastingForm.tsx  # 品鉴表单
│   ├── HistoryPanel.tsx # 历史记录面板
│   └── ShareCard.tsx   # 分享名片组件
├── utils/
│   └── cropImage.ts     # 图片裁剪工具
└── styles/
    └── global.css       # 全局样式
```

### 3.3 数据流
```
用户交互 → useWheelEngine → Zustand Store → 组件响应更新
    ↓
TastingForm保存 → Store更新 → HistoryPanel刷新
    ↓
HistoryPanel点击 → Store读取 → 详情弹窗展示
```

## 5. 非功能需求

### 5.1 性能要求
- 首屏加载时间 ≤ 2秒
- 轮盘交互帧率稳定60fps
- 内存占用 ≤ 100MB
- 打包体积 ≤ 500KB

### 5.2 兼容性要求
- Chrome 90+
- Firefox 88+
- Safari 14+
- Edge 90+

### 5.3 数据安全
- 所有数据本地存储，不上传服务器
- 图片Base64存储
- 支持数据导出

4. 数据持久化正常工作
5. 分享图片生成正确

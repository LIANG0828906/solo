## 1. 架构设计

本项目采用纯前端架构，使用TypeScript + Vite构建，无后端依赖。按职责划分为四个核心模块，通过MainPanel作为控制器进行协调。

```mermaid
    UI -->|用户输入/操作| MainPanel
    MainPanel -->|表达式字符串| Parser
    Parser -->|f(x)函数对象| MainPanel
    MainPanel -->|f(x)+参数| Plotter
    MainPanel -->|f(x)+区间| Analyzer
    Plotter -->|绘图指令| Canvas
    Analyzer -->|分析结果| MainPanel
    MainPanel -->|标记数据| Plotter
```

**模块间数据流向**：
1. MainPanel → Parser（表达式字符串）→ 返回 f(x): (x:number)=>number
2. MainPanel → Plotter（f(x) + ViewRange + StyleOptions）→ 直接绘制至Canvas
3. MainPanel → Analyzer（f(x) + AnalysisRange）→ 返回 AnalysisResult（极值点、拐点、区间）
4. MainPanel → Plotter（AnalysisResult + MarkOptions）→ 在Canvas上叠加标记

## 2. 技术描述

- **前端**：TypeScript 5.x（严格模式，target ES2020）+ 原生HTML/CSS/Canvas API
- **构建工具**：Vite 5.x
- **无第三方图表库**：所有绘图逻辑通过原生Canvas 2D API实现
- **无UI框架**：使用原生DOM API操作界面元素

**选型依据**：
- 用户明确要求"原生JavaScript实现，不依赖第三方图表库"
- Canvas 2D API已足够绘制数学函数、几何标记等基础图形
- Vite提供快速HMR开发体验
- TypeScript严格模式保障数学计算代码的类型安全

## 4. 核心数据结构

### 4.1 类型定义

```typescript
// 视图范围（像素坐标↔数学坐标映射）
interface ViewRange {
  xMin: number; xMax: number;  // X轴数学坐标范围
  yMin: number; yMax: number;  // Y轴数学坐标范围
  xTickStep: number;           // X轴刻度步长
  yTickStep: number;           // Y轴刻度步长
}

// 绘图样式选项
interface StyleOptions {
  curveColor: string;          // 曲线颜色
  showGrid: boolean;           // 是否显示网格
  showAxisLabels: boolean;     // 是否显示坐标轴标签
  gridDensity: number;         // 网格密度（1=标准，2=密集，0.5=稀疏）
}

// 数学分析结果
interface AnalysisResult {
  extrema: ExtremumPoint[];    // 极值点集合
  inflections: InflectionPoint[]; // 拐点集合
  monotonicity: Interval[];    // 单调区间
  concavity: Interval[];       // 凹凸区间
}

// 极值点
interface ExtremumPoint {
  x: number; y: number;
  type: 'maximum' | 'minimum';
}

// 拐点
interface InflectionPoint {
  x: number; y: number;
}

// 带性质的区间
interface Interval {
  start: number; end: number;
  property: 'increasing' | 'decreasing' | 'convex' | 'concave';
}
```

### 4.2 Parser 核心算法
- 词法分析：将字符串拆分为 Token（数字、变量x、运算符、函数名、括号）
- 语法分析：递归下降法构建表达式AST（运算符优先级：^ > 一元负 > * / > + -）
- 代码生成：遍历AST生成等价的JavaScript函数体字符串，使用 `new Function()` 编译

### 4.3 Analyzer 核心算法（数值法）
```
一阶导数 f'(x) ≈ [f(x+h) - f(x-h)] / (2h)   // 中心差分，h=1e-5
二阶导数 f''(x) ≈ [f(x+h) - 2f(x) + f(x-h)] / h²

极值点检测：遍历网格（步长0.01），检测 f'(x) 变号且 f''(x)≠0
拐点检测：遍历网格，检测 f''(x) 变号
区间划分：根据导数正负变化点划分区间，标记性质
```

### 4.4 Plotter 坐标映射
```
数学坐标(x,y) → 画布坐标(px,py):
  px = paddingLeft + (x - xMin) / (xMax - xMin) * (width - paddingLeft - paddingRight)
  py = height - paddingBottom - (y - yMin) / (yMax - yMin) * (height - paddingTop - paddingBottom)
```

### 4.5 性能保障策略
- **曲线绘制**：步长0.1沿X轴采样，超出视图范围时裁剪
- **重绘优化**：每次clearRect后只绘制必要元素，拖动时使用requestAnimationFrame节流
- **分析计算**：将 [-10,10] 切分为2000个采样点（步长0.01），单次遍历完成全部检测
- **内存管理**：避免频繁创建大数组，分析结果复用临时数组

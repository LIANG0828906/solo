## 1. 产品概述

植物生长记录与智能照料提醒应用，帮助用户管理家中植物的照料日程。用户可以为每盆植物创建档案，记录浇水、施肥、换盆等事件，系统根据植物种类自动计算并推送下次照料提醒。

- **主要用途**：植物档案管理、照料事件记录、智能提醒推送
- **目标用户**：家庭植物爱好者、绿植新手
- **核心价值**：通过智能化提醒帮助用户科学照料植物，减少植物因忘记浇水/施肥而枯萎的情况

## 2. 核心功能

### 2.1 功能模块

1. **主页（花园）**：植物卡片网格展示、种类筛选、待办提醒区
2. **植物详情页**：植物大图展示、基本信息、照料时间线、快速记录面板
3. **添加/编辑植物**：植物表单（名称、种类、购买日期、位置、图片上传）

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|-----------|-------------|---------------------|
| 主页 | 待办提醒区 | 展示48小时内到期或已逾期的照料任务，点击跳转详情页 |
| 主页 | 种类筛选栏 | 圆角标签按钮，按植物种类筛选卡片 |
| 主页 | 花园网格 | 植物卡片网格，交错淡入动画，悬浮阴影效果 |
| 详情页 | 植物信息区 | 大图展示、名称、种类、位置、购买日期 |
| 详情页 | 照料时间线 | 垂直时间线，倒序展示所有事件，可删除事件 |
| 详情页 | 快速记录面板 | 一键记录浇水/施肥/换盆/修剪事件，自动更新下次提醒 |
| 添加页 | 植物表单 | 填写植物信息，上传封面图片 |

## 3. 核心流程

### 3.1 用户主流程

用户打开应用 → 查看待办提醒 → 点击提醒进入植物详情 → 记录照料事件 → 时间线更新并刷新下次提醒时间

### 3.2 植物管理流程

```mermaid
flowchart LR
    A["主页"] --> B["点击添加植物"]
    B --> C["填写植物表单"]
    C --> D["提交保存"]
    D --> E["植物卡片出现在花园"]
    E --> F["点击卡片进入详情"]
    F --> G["查看时间线/记录事件"]
```

### 3.3 提醒计算逻辑

- 浇水间隔：多肉7天、绿萝3天、兰花5天、其他植物4天
- 施肥间隔：统一30天
- 换盆/修剪：无固定间隔，记录后不生成提醒
- 提醒状态：已过期（红色）、即将到期48小时内（黄色）、正常（绿色）

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：自然柔和的绿色系 #4CAF50
- **背景色**：浅米色 #FFF8E1
- **卡片样式**：白色圆角矩形（16px圆角），柔和阴影，悬浮时阴影加深并上移4px
- **字体**：现代无衬线字体，清晰易读
- **图标风格**：SVG图标，时间线节点带脉冲光晕动画

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|-----------|-------------|-------------|
| 主页 | 待办提醒区 | 卡片列表、红色逾期标签、黄色即将到期标签 |
| 主页 | 种类筛选栏 | 圆角标签按钮、选中态高亮 |
| 主页 | 花园网格 | 响应式网格（桌面3列/平板2列/手机1列）、交错淡入动画 |
| 详情页 | 植物信息区 | 大图、名称标题、基本信息标签 |
| 详情页 | 照料时间线 | 垂直时间线、SVG事件图标、脉冲动画、淡入滑入效果 |
| 详情页 | 快速记录面板 | 按钮涟漪扩散动效、事件类型图标 |

### 4.3 响应式设计

- 桌面端（≥1024px）：花园网格3列
- 平板端（768px-1023px）：花园网格2列
- 移动端（<768px）：花园网格1列，布局垂直堆叠

### 4.4 动效设计

- 页面切换：0.3秒淡入/淡出过渡
- 卡片悬浮：阴影加深 + 上移4px
- 时间线新节点：淡入 + 向上滑入动画
- 按钮点击：涟漪扩散动效
- 种类切换：卡片交错逐张淡入
- 时间线图标：脉冲光晕动画

## 5. 性能要求

- 页面初始化加载时间 ≤ 1.5秒
- 时间线滚动流畅（60fps）
- 添加事件后时间线更新响应时间 < 200ms
- 图片加载优化（使用合适的图片格式和尺寸）

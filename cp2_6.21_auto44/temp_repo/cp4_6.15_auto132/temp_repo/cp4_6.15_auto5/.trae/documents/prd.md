## 2. 产品概述

### 2.1 背景与痛点

独立开发者和小型团队在运营个人网站或微服务时，面临以下痛点：
- **手动检查效率低**：需要手动访问或 curl 检查服务是否可用，无法做到实时感知
- **缺乏统一面板**：多个服务分散管理，没有集中化的可视化健康状态看板
- **告警机制缺失**：服务宕机后无法第一时间收到通知，发现故障时往往已过去很久
- **历史回溯困难**：难以追踪过去何时发生过故障、持续了多久、影响范围如何

### 2.2 产品目标

构建一个纯前端、零部署成本的轻量级监控仪表板，用户只需打开浏览器即可：
- 实时监控多个 HTTP/HTTPS 服务端点的健康状态
- 通过可视化卡片快速感知异常
- 故障发生时即时收到浏览器桌面通知
- 随时回溯历史故障记录与统计指标

### 2.3 目标用户

- 独立开发者 / 自由职业者
- 2-5 人小型技术团队
- 需要监控 5-30 个服务端点的运维人员

## 4. 非功能需求

### 4.1 视觉设计
- 深色主题配色：
  - 主背景：`#1a1a2e`
  - 卡片底色：`#16213e`
  - 选中高亮：`#0f3460`
  - 警告色：`#e94560`
  - 文字色：`#a8b2d1`
- 整体风格：科技感、仪表盘风格、精致微交互

### 4.2 性能要求
- 端点数量 ≤ 20 时，主线程帧率 ≥ 55fps
- 页面切换 / 滚动无明显卡顿或闪烁
- 所有 fetch 请求使用 AbortController 超时控制
- 组件使用 React.memo 合理优化重渲染

### 4.3 技术约束
- 技术栈：TypeScript + React + Vite
- 纯前端运行，无后端依赖
- 数据持久化：localStorage
- 两个核心模块：`src/engine/`（监控引擎）+ `src/ui/`（UI展示）
- 模块间通过 `src/store/`（状态中心）通信，禁止直接耦合

## 6. 数据模型（核心）

### 6.1 MonitoredEndpoint（端点配置）
```typescript
{
  id: string;              // UUID
  name: string;            // 显示名称
  url: string;             // 监控 URL
  interval: number;        // 检测间隔（秒）
  timeout: number;         // 超时时间（秒）
  expectedStatuses: number[]; // 期望状态码
  failureThreshold: number;   // 连续失败阈值
  createdAt: number;       // 创建时间戳
}
```

### 6.2 CheckResult（单次检测结果）
```typescript
{
  endpointId: string;
  timestamp: number;       // 检测时间戳
  statusCode: number | null; // HTTP状态码（超时/网络错误为null）
  latency: number;         // 响应延迟（毫秒）
  isTimeout: boolean;      // 是否超时
  isSuccess: boolean;      // 是否成功（状态码在期望值内）
  errorMessage?: string;   // 错误信息
}
```

### 6.3 FailureEvent（故障事件）
```typescript
{
  id: string;
  endpointId: string;
  startTime: number;       // 故障开始时间
  endTime: number | null;  // 故障结束时间（null=仍在故障中）
  duration: number | null; // 持续时长（毫秒）
  checkResults: CheckResult[]; // 故障期间所有检测记录
  isResolved: boolean;     // 是否已恢复
}
```

## 7. 验收标准

| 编号 | 验收条件 |
|------|----------|
| A1 | 添加3个端点，配置不同间隔，各端点按预期频率独立检测 |
| A2 | 模拟端点返回非200状态码，连续失败3次后触发告警通知 |
| A3 | 卡片状态灯颜色与实际健康状态一致，过渡动画平滑 |
| A4 | 导出配置后清空 localStorage，导入配置后端点完整恢复 |
| A5 | 故障历史页面可正确筛选并展示过往故障记录 |
| A6 | 20个端点同时检测时，Chrome DevTools Performance 面板显示帧率≥55 |
| A7 | 窗口宽度<768px时，布局自动切换为单列+摘要置顶 |
| A8 | 浏览器未授权通知时，右上角提醒条正确显示 |

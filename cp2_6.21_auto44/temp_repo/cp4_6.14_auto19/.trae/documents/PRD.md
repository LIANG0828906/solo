# 企业团队内部活动投票与决策工具 - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定位
面向企业团队内部使用的轻量级活动投票与决策工具，帮助团队高效、透明地完成各类集体决策。

### 1.2 目标用户
- 团队负责人 / 项目经理（投票发起者）
- 团队成员（投票参与者）

### 1.3 核心价值
- **高效决策**：快速发起投票，缩短团队决策周期
- **实时透明**：实时展示投票趋势，结果一目了然
- **流程自动化**：截止时间管理、结果生成、分享图一站式完成

---

## 2. 功能需求

### 2.1 投票创建模块
| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 表单输入 | 支持输入投票标题（必填）、描述（选填） | P0 |
| 选项管理 | 支持2-10个选项的动态增删 | P0 |
| 截止时间 | 设置投票截止时间（选填） | P0 |
| 投票码生成 | 创建成功后自动生成6位字母数字混合的唯一投票码 | P0 |
| 一键复制 | 投票码支持一键复制功能 | P0 |
| 结果预览 | 创建后跳转到投票结果预览页 | P0 |

### 2.2 投票参与模块
| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 投票码输入 | 参与者输入投票码进入投票页面 | P0 |
| 投票码校验 | 校验投票码有效性，无效给出提示 | P0 |
| 防重复投票 | 每个IP地址或设备标识限制只能投一次票 | P0 |
| 交互动画 | 点击选项卡片有按下缩放动画（scale(0.95) + 200ms transition） | P0 |
| 选中状态 | 选中选项显示对勾图标 + 绿色渐变高亮 | P0 |
| 立即生效 | 点击选项立即记录投票，无需二次确认 | P0 |

### 2.3 结果展示模块
| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| Canvas条形图 | 横向条形图展示各选项得票数和百分比 | P0 |
| 入场动画 | 条形图逐条出现（间隔100ms，从左向右增长，ease-out） | P0 |
| 最高票标记 | 得票数最高的选项顶部显示金色皇冠图标 | P0 |
| 实时更新 | 投票提交后结果面板实时刷新 | P0 |

### 2.4 投票管理模块
| 功能项 | 描述 | 优先级 |
|--------|------|--------|
| 结束投票 | 投票创建者可提前结束投票 | P1 |
| 结果固化 | 结束后结果面板固定不再变化 | P1 |
| 分享图生成 | 自动生成Canvas绘制的结果分享图片（含标题、条形图、最高票标记） | P1 |
| 图片保存 | 支持右键保存分享图为PNG | P1 |

## 3. 非功能需求

### 3.1 视觉设计
- **主题色**：蓝橙撞色
  - 主背景：`#0f172a`
  - 卡片背景：`#1e293b`
  - 按钮主色：`#f97316`
  - 强调色：`#3b82f6`
- **按钮悬停效果**：向上浮动1px + 阴影加深 `box-shadow: 0 4px 12px rgba(249,115,22,0.3)`
- **输入框聚焦**：底部蓝色呼吸光晕动画（2s infinite alternate）
- **响应式适配**：移动端卡片两列变一列，条形图宽度自适应

### 3.2 性能指标
| 指标 | 目标值 |
|------|--------|
| 条形图渲染时间 | ≤ 50ms（100个投票项以内） |
| 投票结果刷新延迟 | ≤ 200ms |
| 首屏加载 | ≤ 2s |

### 3.3 技术约束
- 框架：React 18 + TypeScript
- 构建工具：Vite
- 状态管理：React Context / useState
- 动画：CSS Transition + Canvas Animation
- 数据持久化：LocalStorage（前端模拟后端）

## 4. 用户流程

### 4.1 投票创建流程
```
首页 → 点击"创建投票" → 填写表单（标题/描述/选项/截止时间）
→ 提交 → 生成投票码 → 展示结果预览页（可分享投票码）
```

### 4.2 投票参与流程
```
首页 → 输入投票码 → 校验通过 → 展示投票选项
→ 点击选项（记录投票）→ 实时查看结果面板
```

### 4.3 结果查看流程
```
结果页 → 实时查看条形图 → （可选）结束投票
→ 生成分享图 → 右键保存PNG
```

## 5. 数据结构

### 5.1 投票数据模型
```typescript
interface Poll {
  id: string;           // 唯一投票码（6位字母数字）
  title: string;        // 投票标题
  description: string;  // 投票描述
  options: PollOption[]; // 选项列表
  deadline: number | null; // 截止时间戳
  createdAt: number;    // 创建时间戳
  endedAt: number | null; // 结束时间戳
  isEnded: boolean;     // 是否已结束
  creatorDeviceId: string; // 创建者设备标识
}

interface PollOption {
  id: string;
  text: string;
  votes: number;
}

interface VoteRecord {
  pollId: string;
  optionId: string;
  deviceId: string;
  timestamp: number;
}
```

## 6. 页面结构

1. **首页（Home）**：创建投票入口 + 投票码输入入口
2. **创建投票页（CreatePoll）**：表单填写区域
3. **投票页（VotePage）**：投票选项展示 + 实时结果面板
4. **结果页（ResultsPanel）**：Canvas条形图 + 分享图 + 结束投票按钮

---

## 7. 验收标准

- [ ] 可成功创建2-10个选项的投票
- [ ] 创建后生成6位字母数字混合投票码，支持一键复制
- [ ] 通过投票码可正确进入投票页面
- [ ] 同一设备/IP只能投一次票
- [ ] 选项卡片点击有缩放动画和选中高亮
- [ ] 条形图正确展示得票数和百分比，有入场动画
- [ ] 最高票选项显示金色皇冠
- [ ] 投票结果在200ms内刷新
- [ ] 条形图渲染不超过50ms
- [ ] 可结束投票并生成分享图
- [ ] 界面适配移动端

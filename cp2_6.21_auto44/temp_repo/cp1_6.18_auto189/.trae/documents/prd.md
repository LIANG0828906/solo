## 1. 产品概述

搭配预览家是一款面向线上家具品牌客户的全栈Web应用，允许用户上传自家房间照片并在照片上自由摆放品牌家具3D模型，预览实际搭配效果，从而降低退货率并提升购买意愿。

- 核心价值：让客户在购买前直观感受家具在自家空间中的搭配效果，消除"买回家不搭配"的顾虑
- 目标用户：线上家具品牌的潜在购买客户

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册 | 上传房间照片、浏览家具库、放置调整家具、保存分享场景 |
| 分享链接访问者 | 无需注册 | 查看他人保存的搭配场景 |

### 2.2 功能模块

1. **主页面**：图片上传区域、房间照片展示、3D家具放置场景、家具库面板、底部工具栏
2. **分享页面**：展示保存的搭配场景（只读）

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主页面 | 图片上传区 | 拖拽或点击上传房间照片（JPG/PNG，最大5MB），上传成功后照片全幅显示，背景深色遮罩，照片边缘发光边框 |
| 主页面 | 家具库面板 | 右侧面板（280px宽），6种预设家具模型缩略图卡片（80x80px），点击加载3D模型到照片默认位置 |
| 主页面 | 3D放置场景 | Three.js场景叠加在照片上，支持鼠标拖拽移动家具、滚轮缩放、Delete删除、ESC取消选中 |
| 主页面 | 底部工具栏 | 固定底部，保存按钮将场景数据发送到后端，返回分享链接 |
| 分享页面 | 场景展示 | 根据分享UUID加载并展示保存的搭配场景 |

## 3. 核心流程

用户打开应用 → 上传房间照片 → 照片全幅显示在页面中央 → 从右侧家具库选择家具 → 3D模型出现在照片默认位置 → 拖拽调整位置和缩放 → 保存场景获取分享链接 → 分享给他人查看

## 4. 用户界面设计

### 4.1 设计风格

- 主题色：深色系 — 主背景#1A1A2E，侧边栏#16213E，强调色#0F3460
- 辅助色：暖木色#D4A76A（家具模型色），发光边框#E0B060，保存按钮#6C5CE7
- 按钮风格：圆角、悬停上升动画、点击press状态（scale 0.95）
- 字体：使用优雅的衬线体搭配现代无衬线体
- 布局风格：中央大画布 + 右侧毛玻璃面板 + 底部固定工具栏
- 图标风格：线性图标（lucide-react）

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 主页面 | 图片上传区 | 深色遮罩背景rgba(0,0,0,0.7)，照片发光边框#E0B060 2px，拖拽上传区域 |
| 主页面 | 家具库面板 | 毛玻璃效果rgba(45,45,68,0.6) blur 10px，家具缩略图80x80px圆角8px背景#2D2D44，悬停缩放1.05 |
| 主页面 | 3D放置场景 | 半透明Three.js场景zIndex 10，家具模型哑光质感#D4A76A，柔和阴影1024x1024 |
| 主页面 | 底部工具栏 | 背景rgba(30,30,60,0.8)高度60px圆角12px 12px 0 0，磁盘图标#6C5CE7，悬停上升4px |
| 分享页面 | 场景展示 | 只读模式展示保存的搭配场景，背景风格与主页一致 |

### 4.3 响应式适配

- 桌面优先设计
- 窗口宽度 < 768px时，家具库面板折叠为底部可展开抽屉（从下往上滑入，动画0.3s cubic-bezier(0.4, 0, 0.2, 1)）
- 3D场景区域自适应窗口尺寸

### 4.4 3D场景指引

- 环境：暗色空间，无环境贴图，照片作为背景层
- 灯光：方向光产生柔和阴影，阴影贴图分辨率1024x1024
- 相机：正交相机，与照片平面对齐
- 交互：鼠标拖拽移动家具、滚轮缩放（范围0.5-2.0，步长0.1）、选中发光轮廓动画
- 家具模型：使用程序化几何体模拟低多边形家具（面数≤2000），哑光材质#D4A76A
- 缩放辅助线：模型缩放时出现半透明圆环
- 性能预算：模型加载 < 1秒，交互帧率 ≥ 45FPS，场景初始化 < 500ms

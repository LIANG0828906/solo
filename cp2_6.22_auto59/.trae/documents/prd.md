## 1. 产品概述

NutriTrack 是一款智能个人营养摄入与饮食计划管理应用，旨在解决用户日常饮食记录繁琐、缺乏营养数据跟踪与健康建议的问题。目标用户为注重健康饮食的都市人群，通过食物数据库驱动的智能记录、个性化营养目标设定和历史趋势分析，帮助用户科学管理每日饮食营养。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册（单用户模式） | 浏览与使用全部功能 |

### 2.2 功能模块

1. **主页仪表盘**：饮食日志列表、营养环形图、进度条、今日推荐、快捷操作
2. **食物搜索与添加页**：搜索食物、查看营养信息、添加到当日日志
3. **设置页**：填写基础信息、计算BMR、设定营养目标
4. **饮食报告页**：7天趋势图表、历史记录查看与编辑

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主页仪表盘 | 饮食日志列表 | 显示当日已添加食物记录，支持按餐次分类（早餐/午餐/晚餐/加餐），每条记录显示食物名称与营养摘要，支持删除操作，添加时卡片从底部滑入动画 |
| 主页仪表盘 | 营养环形图 | 实时展示当日总营养摄入（蛋白质/脂肪/碳水/膳食纤维），不同颜色区分营养素，添加食物时平滑更新动画 |
| 主页仪表盘 | 营养目标进度条 | 顶部展示热量和三大营养素目标进度，摄入>90%变橙色，>100%变红色并闪烁 |
| 主页仪表盘 | 今日推荐卡片 | 傍晚时段根据当日摄入推荐补充菜肴，卡片从右侧滑入，带阴影和渐变背景 |
| 主页仪表盘 | 智能提醒横幅 | 每日上午9点和下午3点推送未完成饮食记录提醒，伴随抖动动画 |
| 食物搜索页 | 搜索框 | 输入食物名称搜索，从内置数据库返回匹配项 |
| 食物搜索页 | 搜索结果列表 | 每项显示热量、蛋白质、脂肪、碳水、膳食纤维，点击选择添加到日志 |
| 设置页 | 基础信息表单 | 填写身高、体重、年龄、性别、活动水平 |
| 设置页 | BMR计算与目标推荐 | 根据Harris-Benedict公式计算基础代谢率，推荐每日热量和三大营养素目标，可手动微调 |
| 饮食报告页 | 7天趋势图 | 过去7天每日总热量折线图，平均蛋白质和碳水摄入柱状图对比，鼠标悬浮查看具体数值 |
| 饮食报告页 | 历史记录详情 | 点击日期展开该日详细餐次记录，每个餐次可编辑或删除 |

## 3. 核心流程

用户打开应用 → 在主页查看当日营养摄入状态与目标进度 → 通过搜索框搜索食物并添加到日志 → 系统实时更新环形图与进度条 → 设置页面填写个人信息获取推荐目标 → 报告页面查看7天趋势与历史记录 → 系统在特定时段推送提醒与菜肴推荐

## 4. 用户界面设计

### 4.1 设计风格

- 主色：草木绿（#4CAF50），强调色：暖橙（#FF9800），背景色：浅灰白（#F5F7FA）
- 按钮风格：圆角按钮，带0.2秒背景色渐变hover动效，点击时有轻微缩放和涟漪反馈
- 字体：标题使用 Noto Sans SC Bold，正文使用 Noto Sans SC Regular
- 布局风格：固定顶部导航栏，三列响应式卡片布局（左侧40%/中间35%/右侧25%），768px以下单列
- 图标风格：使用 lucide-react 线性图标
- 卡片风格：纯白背景，圆角12px，柔和阴影

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 主页仪表盘 | 顶部导航栏 | 左侧logo+应用名，右侧日期选择器+用户头像，固定定位，白色背景带底部阴影 |
| 主页仪表盘 | 饮食日志列表(左40%) | 白色卡片列表，每条记录带食物图标，添加时底部滑入动画，按餐次分组显示 |
| 主页仪表盘 | 营养环形图(中35%) | Recharts环形图，四色营养素分区，中心显示总热量，平滑更新动画 |
| 主页仪表盘 | 今日推荐(右25%) | 右侧滑入推荐卡片，渐变背景+阴影，快捷操作按钮组 |
| 主页仪表盘 | 进度条 | 热量/蛋白质/脂肪/碳水四条进度条，绿→橙→红渐变状态 |
| 主页仪表盘 | 智能提醒 | 顶部横幅通知，抖动动画，可关闭 |
| 食物搜索页 | 搜索区域 | 顶部搜索输入框，实时搜索结果下拉列表 |
| 设置页 | 表单区域 | 居中表单卡片，输入框+选择器+滑块 |
| 饮食报告页 | 图表区域 | Recharts折线图+柱状图，悬浮tooltip，可展开日期详情 |

### 4.3 响应式设计

- 桌面优先设计，三列布局（40%/35%/25%）
- 768px以下切换为单列纵向排列
- 触摸优化：按钮最小44px点击区域，卡片间距适中

### 4.4 性能要求

- 添加食物记录到界面更新：< 300ms
- 图表渲染：< 500ms
- 动画流畅：60fps，使用CSS transform和opacity

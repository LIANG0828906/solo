# 策略卡牌对战应用 - 产品需求文档（PRD）

## 1. 项目概述

### 1.1 项目背景
本项目是一款在浏览器中运行的策略卡牌对战游戏，旨在解决单人模式下缺乏有深度的AI对手和动态卡牌组合乐趣的问题。

### 1.2 目标用户
- 喜欢策略卡牌游戏的玩家
- 希望在浏览器中快速体验对战乐趣的用户
- 桌面端和横向平板用户

### 1.3 核心价值
- 提供有策略深度的AI对战体验
- 精美的视觉效果和流畅的动画
- 丰富的卡牌组合和战斗玩法

---

## 2. 功能需求

### 2.1 卡牌系统

#### 2.1.1 卡牌拖拽与放置
- 玩家可从手牌区域拖拽卡牌到战场格子
- 拖拽时显示半透明残影和轻微缩放动画
- 落位后卡牌正面朝上翻转入场，带有翻牌动画和粒子效果
- 法力不足时卡牌呈现灰暗状态并显示提示文字

#### 2.1.2 卡牌属性
- 每张卡牌拥有：攻击力、生命值、法力消耗
- 特殊技能类型：连击、嘲讽、治疗
- 卡牌稀有度：普通、稀有、史诗、传说（高稀有度有流光动画）

### 2.2 战场系统

#### 2.2.1 战场布局
- 5列 × 3行网格布局
- 己方区域：底部1行
- 敌方区域：顶部1行
- 中立区域：中间1行

#### 2.2.2 格子交互
- 淡淡的网格线显示
- 鼠标悬停时高亮为浅金色
- 可放置区域有明确视觉提示

### 2.3 法力值系统
- 每回合自动恢复1点法力值
- 法力值上限随回合递增（1-10点）
- 顶部显示法力值水晶（水晶形状，剩余发光，消耗时逐个暗淡）

### 2.4 战斗系统

#### 2.4.1 AI对手
- AI根据当前战局选择最优策略
- 优先攻击低血量目标
- 保持卡牌数量优势
- 自动选择出牌位置和目标

#### 2.4.2 攻防结算
- 客户端即时完成攻防计算
- 显示飘字伤害动画（数字向上飘动并渐隐）
- 受伤时生命条红色闪烁

### 2.5 回合系统
- 玩家回合：可出牌、攻击
- 回合结束后AI自动行动
- 回合数统计

### 2.6 结算系统

#### 2.6.1 战斗统计面板
- 从底部滑动抽屉式弹出，带弹簧回弹效果
- 显示内容：回合数、双方总伤害、击杀数、治疗量

#### 2.6.2 胜负动画
- 胜利：金色粒子向上喷射
- 失败：屏幕边缘渐变为深灰色并缓慢收缩

---

## 3. 界面设计

### 3.1 整体主题
- 深色魔幻主题
- 主背景：星空渐变（深蓝到紫黑渐变）
- 缓慢旋转的星云粒子层

### 3.2 视觉元素
- 卡牌边框：金色描边
- 高稀有度卡牌：流光动画
- 己方卡牌入场：淡入上浮动画
- 所有按钮：圆角矩形，悬停时向上弹起并发光

### 3.3 顶部信息栏
- 双方头像
- 生命值进度条（红绿渐变，受伤时红色闪烁）
- 法力值水晶显示

### 3.4 响应式适配
- 桌面端优先
- 横向平板适配

---

## 4. 性能需求

| 指标 | 要求 |
|------|------|
| 帧率 | ≥ 30fps |
| 拖拽响应延迟 | ≤ 100ms |
| 动画响应延迟 | ≤ 100ms |

---

## 5. 技术约束

- 编程语言：TypeScript
- 前端框架：Vue 3
- 构建工具：Vite
- 状态管理：Pinia
- 运行方式：`npm install && npm run dev`

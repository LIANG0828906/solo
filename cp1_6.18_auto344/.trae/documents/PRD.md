# 校舍回声日志 - 产品需求文档

## 1. 产品概述
### 1.1 项目背景
"校舍回声日志"是一个面向毕业生的校园回忆分享平台，让毕业生能够在虚拟校园地图上标记自己最常去的角落，并留下声音或文字回忆，其他用户可以收听这些回忆，并通过热力图了解各个地点的当前人气热度。

### 1.2 目标用户
- 毕业生群体、校友、在校学生

### 1.3 核心价值
- 为校友提供情感寄托和回忆分享的载体
- 通过地图可视化的方式重现校园生活记忆
- 通过热度热力图展示校园地点的情感价值分布

## 2. 功能需求

### 2.1 首页校园地图
- 俯视风格的校园手绘地图（SVG绘制）
- 浅米色#F5F0E1为底，深棕#5D4037细线勾勒建筑和道路
- 中央有指向标的罗盘装饰，比例尺1:200
- 可标记地点用发光淡黄色圆点（直径14px）
- 鼠标悬停时圆点放大到20px并浮现地点名称
- 点击地点弹出半透明毛玻璃模态框

### 2.2 回忆展示与添加
- 支持文字回忆（300字以内，实时字数统计
- 支持录音回忆（20秒以内，录音脉冲动画
- 提交后粒子上升特效（浅金色闪光，持续2秒

### 2.3 热度热力图
- 根据最近30天访问和留言次数计算热度
- 热度条渐变：冷蓝#1E88E5到暖红#E53935
- 热度值0-100线性映射
- 热度数据缓存10分钟刷新

### 2.4 交互体验
- 毛玻璃卡片风格，圆角12px
- 柔和阴影，悬停阴影加深并上移2px
- 淡入过渡动画0.3s
- 移动端双指缩放地图0.8~2.0
- 所有交互元素scale(1.05)悬停反馈

## 3. 性能需求
- 页面初次加载到地图可交互 ≤ 3秒
- 热度查询API响应 ≤ 200ms

## 4. 技术栈
- 前端：React + TypeScript + Vite + Zustand
- 后端：Python FastAPI + SQLAlchemy + SQLite
- 状态管理：Zustand
- HTTP客户端：Axios

## 5. 界面设计规范

### 5.1 配色方案
| 用途 | 颜色值 |
|------|--------|
| 主色（暖米） | #F5F0E1 |
| 辅色（校园绿） | #4CAF50 |
| 深棕 | #5D4037 |
| 文字色 | #3E2723 |
| 圆点发光 | 淡黄色 |
| 冷蓝 | #1E88E5 |
| 暖红 | #E53935 |

### 5.2 字体规范
- 地点名称：14px轻量字体，#3E2723
- 全局过渡：0.2s ease-out

### 5.3 阴影规范
- 默认：0 4px 12px rgba(0,0,0,0.1)
- 悬停：0 6px 18px rgba(0,0,0,0.15)

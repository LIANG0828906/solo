## 1. 产品概述

PixelDash 是一款复古像素风格的 2D 自动跑酷游戏，玩家操控蓝色方块角色在自动滚动的场景中跳跃、冲刺来躲避陷阱和收集金币。目标用户为休闲游戏玩家和像素风格爱好者，核心价值在于流畅的操作手感和丰富的视觉反馈。

## 2. 核心功能

### 2.1 功能模块

1. **游戏主界面**: 游戏标题"PixelDash"、开始提示、像素风格视觉
2. **游戏场景**: 自动滚动地形、角色移动与物理、金币收集、陷阱躲避
3. **游戏结束界面**: 最终得分展示、重新开始提示

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 游戏主界面 | 标题区 | 显示"PixelDash"白色48px像素风格阴影标题 |
| 游戏主界面 | 提示区 | "按空格开始"闪烁动画（周期1秒） |
| 游戏场景 | 角色控制 | 蓝色20x20方块，自动右移，跳跃、冲刺操作 |
| 游戏场景 | 地形系统 | 随机平台生成，平滑高度过渡，空隙设计 |
| 游戏场景 | 金币系统 | 金币出现、旋转动画、收集动画、计分 |
| 游戏场景 | 陷阱系统 | 尖刺生成、碰撞检测、死亡动画、红色闪屏 |
| 游戏场景 | 背景层 | 渐变天空、远处山丘剪影缓慢漂移 |
| 游戏场景 | HUD | 右上角实时得分显示，白色24px粗体带拖影 |
| 游戏结束界面 | 结果展示 | "游戏结束"文字、最终得分、按R重开 |

## 3. 核心流程

玩家进入游戏后看到开始界面，按空格开始游戏。角色自动向右移动，玩家通过空格/↑键跳跃躲避空隙和陷阱，通过Shift键冲刺加速，同时收集金币获得分数。碰到尖刺或掉落底部则游戏结束，显示最终得分，按R键重新开始。

## 4. 用户界面设计

### 4.1 设计风格

- 主背景色: #2D2D2D，页面填充色: #1A1A1A
- 主文字色: #FFFFFF，辅助色: #4A7C59（绿色）、#E74C3C（红色）
- 天空渐变: #2C3E50 到 #3498DB
- 所有物体带1px黑色描边增强像素感
- 像素风格阴影效果
- 画布尺寸 800x400px，居中布局

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 游戏主界面 | 标题 | "PixelDash"白色48px，像素风格阴影，居中 |
| 游戏主界面 | 提示文字 | "按空格开始"白色，闪烁动画1秒周期 |
| 游戏场景 | 角色 | 20x20蓝色方块(#4A90D9)，1px黑色描边 |
| 游戏场景 | 平台 | 宽80-200px，高20px，颜色#4A7C59，1px黑色描边 |
| 游戏场景 | 金币 | 半径10px，颜色#FFD700，旋转动画1.5秒周期 |
| 游戏场景 | 尖刺 | 三角形底宽30px高20px，颜色#E74C3C |
| 游戏场景 | 得分 | 右上角白色24px粗体，带拖影效果 |
| 游戏场景 | 背景 | 渐变天空+深灰色山丘剪影(不透明0.5)缓慢漂移 |
| 游戏结束界面 | 文字 | "游戏结束"+ 最终得分 |

### 4.3 响应式设计

- 画布比例保持不变，宽度自适应屏幕（最大800px）
- 移动设备触摸屏幕右侧区域控制跳跃
- 超出画布区域用#1A1A1A填色

### 4.4 动画效果

- 金币旋转动画（1.5秒周期），收集时缩放+消失动画（0.2秒，1→1.5倍放大并透明）
- 跳跃拖尾：3个半透明残影，透明度递减0.2，持续0.1秒
- 冲刺速度线：随机短横线从右向左闪过，长15-25px，#FFFFFF透明0.3
- 冲刺角色变半透明红色
- 尖刺出现前0.5秒红色闪烁警示
- 死亡屏幕闪红（0.3秒渐隐）
- 得分拖影效果

# 太空资源采集与舰队管理沙盒 - 产品需求文档

## 1. 产品概述
一款基于PixiJS的2D太空策略沙盒游戏，玩家在100x100的网格宇宙中管理AI舰队，自动采集资源、躲避障碍、升级基地，并与太空海盗战斗。

### 1.1 产品定位
交互式太空策略模拟游戏，强调自动化AI调度与视觉表现。

### 1.2 目标用户
策略游戏爱好者、对AI调度算法感兴趣的开发者与玩家。

---

## 2. 核心功能需求

### 2.1 资源场生成系统
- **地图规模**: 100×100 网格
- **资源类型**:
  - 铁矿（橙色）：基础建造材料
  - 水晶（蓝色）：高级升级材料
  - 气体（绿色）：能源材料
- **资源场属性**:
  - 总储量
  - 采集效率
  - 剩余量
- **动态特性**: 资源随采集逐渐缩小，耗尽后播放消失动画并从地图移除

### 2.2 AI舰队调度系统
- **舰队组成**: 每支舰队由 1-5 艘飞船组成
- **核心能力**:
  - A* 路径规划算法
  - 障碍物避让
  - 舰队间碰撞避免
  - 自动采集资源
  - 自动返回基地卸货
- **视觉表现**:
  - 平滑移动动画
  - 速度系数显示
  - 三角形舰队图标
  - 颜色渐变粒子拖尾

### 2.3 基地升级系统
- **等级范围**: 1-5 级
- **升级效果**:
  - 仓库容量提升
  - 舰队建造速度加快
  - 外观变化
  - 影响范围扩大（周边格子高亮显示）
- **升级消耗**: 特定资源组合（铁矿+水晶+气体按比例）
- **视觉效果**: 升级时六边形向外扩散波纹光效

### 2.4 战斗遭遇机制
- **触发条件**: 舰队前往资源场途中随机遭遇太空海盗
- **战斗属性**:
  - 舰队火力值
  - 舰队护甲值
  - 敌方AI属性
- **战斗结果**:
  - 胜利：继续任务
  - 失败：返回基地，损失部分携带资源
- **视觉表现**:
  - 战斗动画
  - 伤害数字跳字效果

### 2.5 实时统计面板
- **统计指标**:
  - 总资源采集量（分类统计）
  - 活跃舰队数
  - 基地总数
  - 海盗击退次数
  - 运行时间
- **更新频率**: 每帧更新
- **动画效果**: 数字滚动过渡动画

## 3. 界面设计规范

### 3.1 整体风格
**科幻太空风格**
- 背景：深空渐变（深紫 → 墨蓝）
- 星星粒子：静态随机分布
- 网格线：半透明发光效果

### 3.2 视觉元素规范
| 元素 | 样式 |
|------|------|
| 铁矿资源场 | 橙色发光六边形，脉冲缩放动画 |
| 水晶资源场 | 蓝色发光六边形，脉冲缩放动画 |
| 气体资源场 | 绿色发光六边形，脉冲缩放动画 |
| 舰队 | 三角形图标，移动时带渐变粒子拖尾 |
| 基地 | 六边形图标，升级时波纹光效扩散 |
| UI面板 | 毛玻璃半透明，霓虹蓝描边 |

### 3.3 布局结构
```
┌─────────────────┬───────────────────────┬──────────────┐
│                 │                       │              │
│  舰队控制栏     │                       │  地图预览    │
│                 │                       │  缩略图      │
│  基地列表       │      主画布           │              │
│  (可滚动)       │   (拖拽+滚轮缩放)     │  统计面板    │
│                 │                       │              │
│                 │                       │  (右下角)    │
└─────────────────┴───────────────────────┴──────────────┘
```

### 3.4 交互细节
- 卡片 hover：上浮 + 发光效果
- 数值变化：平滑过渡动画
- 主画布：拖拽平移、滚轮缩放

---

## 4. 性能需求
- **帧率目标**: 稳定 60fps
- **最大负载**:
  - 50 个资源场
  - 10 个基地
  - 20 艘舰队（共最多100艘飞船）
- **优化策略**:
  - 对象池管理粒子
  - 视口裁剪渲染
  - 路径缓存复用

---

## 5. 技术栈
- **语言**: TypeScript（严格模式）
- **渲染引擎**: PixiJS v7
- **构建工具**: Vite
- **模块系统**: ESNext

---

## 6. 非功能需求
- 响应式：适配主流桌面分辨率（1280×720 及以上）
- 可维护性：模块化架构，单一职责原则
- 可扩展性：预留新资源类型、舰队AI策略的扩展接口

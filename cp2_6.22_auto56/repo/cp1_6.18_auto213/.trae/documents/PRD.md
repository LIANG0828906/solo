# 文明漫游者 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
在线教育平台需要为学生提供一种沉浸式的古代文明历史探索体验，通过交互式时间轴和3D文物展示来替代传统的图文学习方式，提升学生的学习兴趣和知识 retention。

### 1.2 项目目标
- 构建一个沉浸式的历史学习平台
- 通过3D可视化让历史文物"活"起来
- 提供流畅的交互体验和知识关联性
- 保持60FPS的流畅性能

### 1.3 目标用户
- 中学生、大学生历史学习者
- 历史爱好者
- 对古代文明感兴趣的普通用户

### 1.4 关键成功指标
- 3D场景帧率稳定在60FPS
- 模型加载时间不超过3秒
- 交互响应时间小于100ms
- 支持最小宽度1024px自适应布局

## 2. 功能需求

### 2.1 交互式时间轴 (Timeline)
- **位置**: 页面底部，占页面高度20%（窗口宽度<1280px时为25%）
- **内容**: 8个代表不同古代文明的关键事件节点
- **视觉**:
  - 节点为圆形图标
  - 背景颜色从#4A148C（公元前）渐变到#FF8F00（公元后）
  - 鼠标悬停时节点放大1.2倍并显示简短标题
- **交互**:
  - 点击节点后显示垂直连接线指向3D场景区
  - 连接线动画展开0.5s ease-out
  - 输出选中事件ID给状态管理

### 2.2 3D文物展示 (Exhibition Hall)
- **位置**: 页面主区域，占页面高度70%
- **功能**:
  - 加载对应时代的代表文物glTF模型（共8个）
  - 模型多边形数控制在5000以内
  - 模型自动居中并缓慢自转（Y轴，周期12s）
- **场景**:
  - 背景为深蓝色渐变（#0A0F2E到#1A237E）
  - 地面有半透明网格辅助线
- **交互**:
  - 鼠标拖拽旋转（水平范围-90°到90°，垂直范围-30°到30°）
  - 滚轮缩放（范围0.5到3倍）

### 2.3 文物信息卡片
- **位置**: 3D场景右侧叠加
- **样式**:
  - 毛玻璃效果，背景rgba(13,17,23,0.7)
  - 圆角16px，内边距24px
- **内容**:
  - 文物名称
  - 所属年代
  - 文明类型
  - 200字以内背景介绍
- **动画**:
  - 模型加载完成后从右侧滑入（translateX 50px→0，0.4s ease-out）
  - 关闭按钮在左上角，点击后滑出

### 2.4 加载状态与过渡
- **切换动画**:
  - 旧场景淡出（opacity 1→0，0.3s）
  - 新场景淡入（0.5s）
- **加载指示器**:
  - 旋转的环形加载指示器
  - 颜色#64FFDA，大小40px
  - 加载完成后渐隐

### 2.5 推荐关联
- **位置**: 文物信息卡片底部
- **内容**: 2个关联事件建议按钮
- **样式**:
  - 圆角胶囊样式
  - 背景#1E88E5，悬停#1565C0
  - 文字白色，内边距8px 16px
- **功能**: 点击跳转到对应时间轴节点并加载模型

## 3. 界面设计

### 3.1 整体布局
- **主题**: 暗色调沉浸式设计
- **导航栏**: 顶部半透明（背景rgba(13,17,23,0.8)，高56px）
  - 左侧：应用名称"文明漫游者"
  - 右侧：学习记录和设置图标
- **主区域**:
  - 上方80%：3D场景展示区
  - 下方20%：时间轴面板（背景#1A1A2E，顶部1px分隔线#2A2A4E）

### 3.2 视觉规范
- **字体**: 'Merriweather'系列
- **背景色**: #0D1117
- **文字颜色**: #E0E0E0（浅色为主）
- **高亮色**: #64FFDA、#1E88E5
- **交互反馈**: 所有元素有悬停和点击动画（transition 0.2s ease）

### 3.3 响应式设计
- 最小宽度: 1024px
- 窗口宽度<1280px时，时间轴面板高度调整为25%

## 4. 数据模型

### 4.1 时间轴事件数据
```typescript
interface TimelineEvent {
  id: string;
  year: number;           // 年代（负数表示公元前）
  title: string;          // 事件标题
  description: string;    // 简短描述
  civilization: string;   // 文明类型
  color: string;          // 节点颜色
}
```

### 4.2 文物数据
```typescript
interface Artifact {
  id: string;
  name: string;           // 文物名称
  era: string;            // 所属年代
  civilization: string;   // 文明类型
  modelPath: string;      // 3D模型路径
  description: string;    // 背景介绍（<200字）
  relatedEvents: string[]; // 关联事件ID列表
}
```

## 5. 非功能需求

### 5.1 性能要求
- 3D场景帧率: 60FPS（主流设备）
- 模型加载时间: ≤3秒（模拟数据）
- 内存占用: ≤500MB

### 5.2 兼容性
- 支持Chrome、Firefox、Edge、Safari最新版本
- 需要支持WebGL 2.0

### 5.3 可维护性
- 模块化架构，按职责分离
- 清晰的文件调用关系和数据流向
- TypeScript严格模式

## 1. 产品概述

FitPose - 智能健身姿态评估应用，帮助健身爱好者独自训练时获得专业级动作指导。通过实时摄像头捕捉与 AI 姿态检测技术，自动评估动作标准度，提供可视化反馈，解决无人指导难以自查的痛点。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册，本地存储 | 录制训练、查看报告、管理历史记录 |

### 2.2 功能模块

1. **开始训练**：摄像头实时预览、骨骼检测叠加、录制控制
2. **训练报告**：动作评分、偏差热力图、逐帧回放
3. **历史记录**：按日期分组、记录管理（查看/删除）
4. **设置**：动作选择、参数配置

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 训练页面 | 摄像头预览 | 实时显示摄像头画面（640x480，圆角12px，外发光效果） |
| 训练页面 | 骨骼叠加层 | 半透明3D骨骼检测图层，节点8px圆形，连线3px宽度，30fps刷新率 |
| 训练页面 | 录制按钮 | 直径80px圆形按钮，脉冲光圈动画，点击缩放反馈 |
| 报告页面 | 评分展示 | 72px大数字评分，颜色根据分数渐变 |
| 报告页面 | 偏差热力图 | 身体轮廓图，不同部位色块显示偏差程度 |
| 报告页面 | 逐帧回放 | 时间轴滑块拖动，当前帧骨骼对比图 |
| 历史页面 | 日期分组列表 | 按日期分组展示训练记录 |
| 历史页面 | 记录卡片 | 动作名称、时长、评分，支持查看/删除，删除滑出动画 |

## 3. 核心流程

用户打开应用 → 选择训练动作 → 开始录制（实时骨骼检测叠加）→ 结束录制 → 系统逐帧对比标准骨骼模型 → 生成评分与热力图 → 展示训练报告 → 保存记录至历史

```mermaid

    A["开始训练"] --> B["选择动作类型"]
    B --> C["摄像头实时预览"]
    C --> D["点击录制按钮"]
    D --> E["实时骨骼检测叠加"]
    E --> F["结束录制"]
    F --> G["PoseComparator逐帧对比"]
    G --> H["生成评分与热力图"]
    H --> I["展示训练报告"]
    I --> J["保存到历史记录"]
    J --> K["历史记录列表"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深黑背景 #121218，科技绿 #00FF88，强调红 #FF3366
- **辅助色**：黄色 #FFD700（中等偏差），红色 #FF4500（严重偏差）
- **按钮风格**：圆角8px，悬停背景变亮，点击缩放反馈
- **字体**：现代无衬线字体，清晰易读，数字等宽
- **布局风格**：左侧导航栏 + 主内容区居中，卡片式设计
- **视觉效果**：外发光、半透明叠加、微动效，营造科技健身氛围

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 训练页面 | 导航栏 | 左侧240px宽度，深灰背景，三个功能按钮 |
| 训练页面 | 摄像头区域 | 居中布局，左侧画面右侧骨骼层，垂直居中 |
| 训练页面 | 录制控制 | 底部居中圆形录制按钮，脉冲动画 |
| 报告页面 | 评分区域 | 顶部大数字评分，渐变色 |
| 报告页面 | 热力图区域 | 中部身体轮廓图，色块标注 |
| 报告页面 | 回放控制 | 底部时间轴滑块，自定义样式 |
| 历史页面 | 记录列表 | 日期分组标题，卡片式记录条目 |

### 4.3 响应性

- 桌面端优先设计（最大宽度1200px）
- 中等屏幕自适应缩放
- 不考虑移动端适配

### 4.4 性能要求

- 摄像头捕获到骨骼渲染端到端延迟 ≤ 150ms
- 录制时页面帧率 ≥ 30fps
- 骨骼逐帧对比计算 ≤ 20ms/帧

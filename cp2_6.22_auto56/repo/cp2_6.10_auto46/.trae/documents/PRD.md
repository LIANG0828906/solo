## 1. 产品概述

本产品是一个古代书坊全流程管理系统，解决传统作坊中多个工匠（刻工、印工、装订工）同时参与同一本书的不同工序时，工序流转、工时记录和工钱结算容易错漏和延迟的问题。目标用户为书坊主，核心价值是实现从书稿征集、雕版刻印到书册发行的全流程数字化管理，自动结算工匠工钱。

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方法 | 核心权限 |
|------|----------|----------|
| 书坊主 | 系统预设用户 | 书稿登记、工序指派、工钱结算、工匠管理、数据统计 |

### 2.2 功能模块
1. **首页**：统计仪表盘、书稿列表Tab、工匠看板Tab
2. **书稿管理**：书稿登记卡片、工序进度条、详情展开、工序开始操作
3. **工匠管理**：工匠表格、职业筛选、收入排序
4. **结算系统**：自动工钱计算、结算通知、收入累计

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|---------|----------|----------|
| 首页 | 统计仪表盘 | 展示总书稿数、总支出工钱、平均每书制作天数、已完成发行书数四个核心指标 |
| 首页 | 书稿列表 | 卡片形式展示书稿，包含工序进度条、状态标签、开始工序按钮，点击展开详情 |
| 首页 | 工匠看板 | 表格展示工匠信息，支持职业筛选和收入排序 |
| 书稿登记 | 表单模块 | 录入书名、作者、总字数、是否配图、用纸等级 |
| 工序指派 | 弹窗模块 | 选择工匠、设定预计工时、开始工序计时 |

## 3. 核心流程

### 3.1 书稿生命周期流程
```mermaid
flowchart LR
    A["书稿登记"] --> B["待刻版"]
    B --> C["指派刻工+开始刻版"]
    C --> D["刻版中（倒计时）"]
    D --> E["刻版完成<br/>自动结算刻工工钱"]
    E --> F["待印刷"]
    F --> G["指派印工+开始印刷"]
    G --> H["印刷中（倒计时）"]
    H --> I["印刷完成<br/>自动结算印工工钱"]
    I --> J["待装订"]
    J --> K["指派装订工+开始装订"]
    K --> L["装订中（倒计时）"]
    L --> M["装订完成<br/>自动结算装订工工钱"]
    M --> N["已发行"]
```

### 3.2 用户操作流程
1. 书坊主登录系统 → 查看统计仪表盘
2. 点击"新增书稿" → 填写书稿信息 → 提交
3. 在书稿列表中找到待刻版书稿 → 点击"开始刻版" → 选择刻工、设定工时 → 确认
4. 工序倒计时进行中 → 实时查看剩余时间
5. 工序完成 → 弹出结算通知 → 状态自动流转
6. 重复步骤3-5完成印刷和装订工序
7. 切换到工匠看板 → 筛选/排序查看工匠收入情况

## 4. 用户界面设计

### 4.1 设计风格
- **主色调**：米黄宣纸底色 `#f5e6c8`
- **强调色**：暗红 `#8b3a3a`、墨色 `#2a2a2a`
- **统计卡片颜色**：
  - 总书稿数：`#5b8a5b`
  - 总支出工钱：`#8a6b3b`
  - 平均制作天数：`#3b6b8a`
  - 已发行书数：`#8a3b3b`
- **字体**：竖排书法字体用于书坊名称，正文使用清晰易读的宋体类字体
- **按钮风格**：圆角矩形，暗红边框，悬停时背景变为淡红色
- **卡片风格**：米黄色背景，暗红细边框，悬停时上浮2px、阴影加深
- **图标风格**：简约水墨风格SVG图标

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|---------|----------|--------|
| 首页 | 左侧导航栏 | 竖排"临安陈氏书坊"书法标题、三个水墨风格导航图标 |
| 首页 | 统计仪表盘 | 四个等宽彩色圆角卡片，大字加粗数值，中文标签，响应式折行 |
| 首页 | Tab切换栏 | 书稿列表、工匠看板两个Tab，暗红选中状态 |
| 首页 | 书稿卡片 | 书稿信息、三段式工序进度条、状态标签、开始工序按钮、展开箭头 |
| 首页 | 工匠表格 | 固定表头、筛选下拉框、排序按钮、数据行 |
| 通用 | 结算通知 | 顶部淡入横幅，停留3秒自动消失 |
| 通用 | 工序指派弹窗 | 工匠选择下拉框、工时输入框、确认/取消按钮 |

### 4.3 响应式设计
- 桌面端（≥768px）：左右两栏布局，左侧200px固定宽度，右侧自适应
- 移动端（<768px）：顶部导航栏，统计卡片折为两行，书稿卡片单列排列
- 触摸优化：按钮最小高度44px，点击区域充足

### 4.4 动效设计
- 卡片悬停：上浮2px、阴影加深，0.3s ease-out过渡
- 结算通知：淡入（0.3s）→ 停留3s → 淡出（0.3s）
- 书稿展开详情：高度平滑过渡0.3s
- Tab切换：内容区域淡入淡出0.2s
- 倒计时更新：数字平滑过渡

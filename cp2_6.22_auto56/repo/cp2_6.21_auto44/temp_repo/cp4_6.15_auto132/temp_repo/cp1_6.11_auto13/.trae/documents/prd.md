## 1. 产品概述

基于语义索引的文档段落跳跃式导航工具，帮助用户在浏览复杂文档时通过关键词或自然语言查询快速定位到相关段落，解决传统滚动和搜索方式效率低下的问题。

- 目标用户：技术博客读者、长篇报告审阅者、文档编辑人员
- 核心价值：将文档自动分段并建立倒排索引，实现实时模糊搜索与一键跳转，大幅提升长文档导航效率

## 2. 核心功能

### 2.1 功能模块

1. **编辑面板页**：文档输入区域、行号显示、段落分隔线与编号、滚动监听
2. **导航面板页**：搜索框、段落列表、匹配高亮、虚拟滚动、位置记忆

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 编辑面板 | 文本输入 | 带行号的textarea，支持实时输入文档内容 |
| 编辑面板 | 段落分割 | 按连续两个换行或Markdown标题（##级）自动分割段落 |
| 编辑面板 | 段落标记 | 段落间浅色分隔线，左上角灰色段落编号 |
| 编辑面板 | 滚动监听 | IntersectionObserver监听当前可见段落 |
| 导航面板 | 搜索框 | 放大镜图标SVG，支持关键词/自然语言输入，焦点态亮绿色光晕 |
| 导航面板 | 段落列表 | 显示段落编号+首行摘要（最多20字符），虚拟滚动（>200段落） |
| 导航面板 | 匹配高亮 | 匹配子串用#FFF3CD背景标记，实时模糊匹配 |
| 导航面板 | 跳跃导航 | 点击段落条目平滑滚动编辑面板至对应段落首行（300ms） |
| 导航面板 | 位置记忆 | "回到上次跳跃点"按钮，记住最近3个跳跃位置 |

## 3. 核心流程

用户在编辑面板输入或粘贴文档 → 系统自动分段并建立倒排索引 → 用户在导航面板搜索框输入关键词 → 系统实时模糊匹配并高亮显示结果 → 用户点击匹配项 → 编辑面板平滑滚动到对应段落 → 导航面板记录跳跃位置 → 用户可点击"回到上次跳跃点"返回

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深灰色背景（#1E1E1E）+ 亮绿色强调色（#00C853）
- 段落高亮：淡黄色（#FFF3CD）
- 段落分隔线：1px #333
- 段落编号：#888
- 字体：等宽字体用于编辑器，无衬线字体用于导航面板
- 布局：左右分栏，左侧70%（编辑面板），右侧30%（导航面板）
- 搜索框焦点态：亮绿色边框 + box-shadow: 0 0 8px rgba(0,200,83,0.4)
- 列表项悬停：背景色从#2A2A2A过渡到#3A3A3A，150ms ease-in-out
- 匹配高亮：淡黄色背景 + padding: 0 2px

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 编辑面板 | 文本输入区 | 深色背景textarea，左侧行号栏，段落间#333分隔线，左上角#888段落编号 |
| 导航面板 | 搜索框 | 深色输入框，放大镜SVG图标，焦点态亮绿色边框+光晕 |
| 导航面板 | 段落列表 | 深色列表项，悬停渐变动画，匹配文本淡黄色高亮 |
| 导航面板 | 位置记忆 | 顶部亮绿色"回到上次跳跃点"按钮 |

### 4.3 响应式设计

- 桌面端（≥900px）：左右分栏布局，左侧70%，右侧30%
- 移动端（<900px）：导航面板收为底部抽屉（高度可拖拽），编辑面板占满剩余高度

### 4.4 性能要求

- 100KB文本段落分割和索引构建耗时 ≤ 200ms
- 搜索输入实时响应延迟 < 150ms（200段落）
- 滚动跳跃平滑动画帧率 ≥ 50FPS

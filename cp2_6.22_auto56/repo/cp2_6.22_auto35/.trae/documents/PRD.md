## 1. 产品概述

TrailPlanner是一款面向户外活动爱好者的徒步路线规划与分享平台，解决传统路线规划工具缺乏实时数据、无法预估体力消耗以及队友设备不同步的痛点。通过整合地图绘制、海拔分析、实时位置共享、装备管理和体能建模，为徒步爱好者提供一站式路线规划解决方案。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册，本地存储 | 创建路线、加入活动、查看队友位置、管理装备清单 |

### 2.2 功能模块

1. **地图交互模块**：OpenStreetMap底图、路径绘制、节点拖拽调整、海拔剖面图
2. **实时位置共享模块**：活动创建、6位邀请码、WebSocket实时推送、离线缓存
3. **装备与天气模块**：可勾选装备清单、预设装备库、未来三天天气预报
4. **体能消耗模块**：卡路里实时计算、动态折线图、困难区段高亮

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主页面 | 地图视图 | OpenStreetMap底图渲染，支持点击绘制路径，拖拽调整路径节点 |
| 主页面 | 路线概览卡片 | 显示缩略海拔图、总距离、累计爬升/下降、预计完成时间 |
| 主页面 | 队友列表 | 显示队友头像、状态圆点、最后更新时间 |
| 主页面 | 装备清单 | 可勾选装备列表，支持自定义添加 |
| 主页面 | 天气卡片 | 显示未来三天降雨概率和温度范围 |
| 主页面 | 体能消耗图 | 动态折线图显示卡路里消耗，困难区段高亮标记 |

## 3. 核心流程

用户在地图上点击绘制徒步路线 → 系统自动计算海拔数据、距离、爬升/下降 → 基于Naismith法则预估完成时间 → 用户创建活动生成6位邀请码 → 队友通过邀请码加入 → WebSocket实时同步位置信息 → 用户管理装备清单 → 查看天气预报和体能消耗分析。

```mermaid
    B --> C["计算体能消耗"]
    C --> D["创建活动"]
    D --> E["生成邀请码"]
    E --> F["队友加入"]
    F --> G["实时位置同步"]
    G --> H["装备清单管理"]
    H --> I["天气查询"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：森林绿(#2E7D32)、大地棕(#6D4C41)
- **强调色**：醒目橙色(#FF9800)用于行动按钮和重要数据高亮
- **卡片样式**：圆角8px，微弱阴影，hover时阴影加深并上移2px，过渡动画0.2s
- **字体**：标题使用Montserrat，正文使用Inter，建立清晰的层级对比
- **交互反馈**：拖拽路径节点时有弹性动画，海拔图随节点拖动平滑过渡

### 4.2 页面布局

| 区域 | 模块名称 | UI元素 |
|------|----------|--------|
| 左侧70% | 地图视图 | Leaflet地图、可编辑路径图层、队友标记、路径节点 |
| 右侧30% | 路线概览卡片 | 缩略海拔图、距离、爬升/下降、预计时间 |
| 右侧30% | 队友列表 | 头像、状态圆点(在线/离线)、昵称、距离 |
| 右侧30% | 装备清单 | 勾选框、预设装备、添加按钮 |
| 右侧30% | 天气卡片 | 日期、天气图标、降雨概率、温度范围 |
| 右侧30% | 体能消耗图 | 折线图、坡度>20%区段高亮、卡路里数值 |

### 4.3 响应式设计

- **桌面端**：左右分栏布局，地图占70%，面板占30%
- **移动端(<768px)**：上下布局，面板可折叠/展开，优化触摸交互
- **性能要求**：地图渲染帧率≥30fps，节点≥200时海拔图重绘延迟≤100ms

### 4.4 视觉细节

- 地图上添加微妙的山脉纹理叠加层
- 路径使用橙色描边，困难区段使用红色虚线
- 队友标记使用带脉冲动画的头像
- 海拔图使用森林绿渐变填充
- 卡片背景使用米白色(#FAF8F5)营造户外自然感

# VoteHub 产品需求文档（PRD）

## 1. 项目概述

### 1.1 项目背景
社区运营团队需要一个轻量化工具来收集和管理用户对产品功能的投票建议，让团队能直观看到用户最关心哪些功能。

### 1.2 项目目标
构建 VoteHub 功能投票与建议收集系统，支持用户创建投票、参与投票、搜索排序、以及数据可视化统计。

### 1.3 目标用户
- 社区运营团队成员
- 产品经理
- 终端用户（参与投票）

---

## 2. 功能需求

### 2.1 用户创建投票
- 在投票列表页顶部提供创建投票表单
- 输入框规格：宽度 400px，背景色 #2A2A3D，圆角 8px，内边距 12px，字体颜色 #E0E0E0
- 用户输入投票标题和至少 2 个选项（每个选项一行，最多 6 个）
- 提交后生成新投票记录，包含：唯一 ID、标题、选项列表、投票数（初始为 0）、创建时间戳

### 2.2 用户投票
- 每条投票卡片显示标题和选项列表
- 选项按钮规格：宽 100%，内边距 12px 16px，背景色 #3A3A4D，圆角 8px
- 鼠标悬停时背景变 #4A4A5D，过渡 0.2s ease
- 点击后该选项票数 +1，卡片上总投票数更新
- 禁用重复投票（本地存储记录用户已投的投票 ID 列表）

### 2.3 投票卡片展示
- 列表采用两列网格布局（列间距 24px，行间距 24px）
- 卡片规格：宽 100%，背景色 #1E1E2E，圆角 16px，内边距 20px，阴影 0 4px 12px rgba(0,0,0,0.3)
- 鼠标悬停时上浮 4px，阴影增加至 0 8px 24px rgba(0,0,0,0.4)，过渡 0.3s ease-out
- 标题：字体 18px 加粗颜色 #FFFFFF，下边距 16px
- 选项显示选项名和当前票数（字体 14px 颜色 #B0B0B0）
- 底部显示总投票数和创建时间（字体 12px 颜色 #787878）

### 2.4 投票统计页面
- 顶部标签切换器：柱状图、饼图
- 标签规格：内边距 8px 20px，选中标签底部显示 2px 实线颜色 #6C63FF
- 柱状图：X 轴为投票标题，Y 轴为总票数，柱子颜色 #6C63FF
- 饼图：颜色调色板 #6C63FF, #FF6584, #4ECDC4, #FFD93D, #6BCB77，切块间距 2px
- 图表区域：背景色 #1A1A2E，圆角 12px，内边距 20px，字体颜色 #E0E0E0

### 2.5 投票搜索与排序
- 搜索框：宽度 300px，背景色 #2A2A3D，圆角 20px，内边距 8px 16px，字体颜色 #E0E0E0，placeholder 颜色 #787878
- 按标题关键词实时过滤投票（输入延迟 300ms）
- 排序下拉菜单选项：按时间降序、按票数降序、按时间升序
- 切换时有 0.2s 动画过渡

---

## 3. 界面设计规范

### 3.1 主题
- 整体采用深色主题
- 主背景色：#12121C
- 导航栏背景色：#1A1A2E，高度 60px，固定顶部，圆角 0 0 12px 12px，阴影 0 2px 8px rgba(0,0,0,0.2)

### 3.2 导航栏
- 左侧：应用名 VoteHub（字体 24px 加粗颜色 #FFFFFF，图标 emoji: ✓）
- 右侧：两个导航链接（投票列表、统计页面）
  - 字体 14px 颜色 #B0B0B0
  - 选中颜色 #FFFFFF，下边距 2px 实线颜色 #6C63FF
  - 悬停时颜色变 #E0E0E0

### 3.3 内容区域
- 左右内边距 48px，上下内边距 32px
- 响应式：屏幕宽度 < 768px 时切换为一列布局，内边距缩小至 24px

### 3.4 交互动画
- 所有交互（按钮点击、页面切换、卡片悬浮）都应有 0.2s - 0.3s 的平滑过渡动画

---

## 4. 性能要求

- 投票列表渲染 500 张卡片时，初次渲染时间不超过 1200ms
- 搜索过滤时输入响应延迟不超过 200ms
- 页面切换动画帧率稳定在 55FPS 以上

---

## 5. 技术栈

- 前端框架：React 18 + TypeScript
- 构建工具：Vite
- 状态管理：Zustand
- 图表库：Recharts
- 工具库：uuid
- 数据持久化：localStorage

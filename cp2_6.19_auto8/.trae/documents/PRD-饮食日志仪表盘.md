## 1. 产品概述
本产品是一个在线营养管理平台的饮食日志仪表盘，帮助用户快速记录每日饮食并获得可视化营养反馈。
- 核心功能：用户记录每餐食物种类和份量，系统自动计算并展示当日总热量及三大宏观营养素摄入比例，同时给出是否达标的提示。
- 目标用户：关注健康饮食、需要管理营养摄入的人群。
- 产品价值：帮助用户科学管理饮食，直观了解营养摄入情况，培养健康饮食习惯。

## 2. 核心功能

### 2.1 功能模块
1. **主仪表盘**：餐食输入表单、营养图表展示、日志卡片列表、营养目标设置
2. **餐食记录模块**：食物名称输入、份量输入、餐次选择、提交反馈动画
3. **营养计算模块**：内置食物营养数据库、自动计算热量和三大营养素
4. **数据可视化模块**：圆环图表展示营养素比例、达标状态指示、总热量显示
5. **日志管理模块**：餐食卡片列表、展开查看详情、删除功能

### 2.2 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 主仪表盘 | 餐食输入表单 | 输入食物名称、份量（克）、选择餐次，提交时微震动反馈，新记录淡入动画 |
| 主仪表盘 | 营养圆环图 | 使用recharts渲染，800ms转动动画，中心显示总热量，颜色渐变与主题统一 |
| 主仪表盘 | 达标状态指示 | 各项营养达标状态，达标绿色对勾上浮，未达标红色叉号抖动，300ms过渡动画 |
| 主仪表盘 | 日志卡片列表 | 卡片式展示餐食记录，点击展开查看详细营养素，箭头300ms旋转动画 |
| 主仪表盘 | 营养目标设置 | 设置每日热量上限、蛋白质下限等目标 |

## 3. 核心流程

```mermaid
flowchart LR
    A["用户输入餐食数据"] --> B["查询营养数据库"]
    B --> C["计算热量和营养素"]
    C --> D["添加到当日日志"]
    D --> E["更新圆环图表"]
    E --> F["更新达标状态"]
    F --> G["更新日志列表"]
    H["设置营养目标"] --> F
```

用户在表单中输入食物名称和份量，选择餐次后提交。系统根据内置数据库查询该食物的营养成分，计算出对应份量的热量、蛋白质、脂肪、碳水化合物数值，并添加到当日日志中。仪表盘实时更新圆环图表和达标状态，日志列表以卡片形式展示所有记录，点击可展开查看详情。

## 4. 用户界面设计

### 4.1 设计风格
- **设计基调**：清新健康、简洁专业的浅色主题
- **主色调**：#4CAF50（健康绿），用于按钮、达标指示
- **辅助色**：#FFB74D（暖橙），用于强调热量数字
- **背景色**：#f5f5f5（浅灰），卡片背景为白色
- **卡片样式**：圆角12px，细微阴影
- **按钮交互**：悬停0.2s缩放，点击涟漪效果
- **字体**：无衬线字体，大数字粗体展示

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 主仪表盘 | 整体布局 | 左侧40%表单区，右侧60%图表+日志区，桌面端两栏，移动端堆叠 |
| 主仪表盘 | 餐食输入表单 | 输入框、下拉选择、提交按钮，提交后表单淡入动画 |
| 主仪表盘 | 圆环图表 | 渐变色圆环，800ms转动动画，中心显示总热量 |
| 主仪表盘 | 达标状态 | 绿色对勾上浮动画，红色叉号抖动动画 |
| 主仪表盘 | 日志卡片 | 从右侧滑入动画，点击展开箭头旋转300ms |
| 主仪表盘 | 响应式切换 | 768px以下切换为上下堆叠，渐变透明度过渡 |

### 4.3 响应式设计
- **桌面优先**：默认两栏布局，左侧表单40%，右侧内容60%
- **平板适配**：1024px以下调整比例为35%/65%
- **移动端**：768px以下切换为上下堆叠布局，表单在上，图表在下
- **触摸优化**：按钮最小高度44px，输入框适当增大触摸区域

### 4.4 动效设计
- **页面加载**：元素依次淡入，延迟100ms错落显示
- **表单提交**：微震动反馈（使用navigator.vibrate），新卡片从右侧滑入
- **图表动画**：800ms转动动画，数据更新时平滑过渡
- **达标状态**：达标项上浮5px，未达标项轻微抖动
- **卡片展开**：箭头旋转180度，300ms平滑过渡
- **按钮交互**：hover时scale(1.05)，active时scale(0.98)，点击涟漪效果

## 1. 产品概述

色彩对比度检测器是一款面向设计师和前端开发者的无障碍配色辅助工具，基于WCAG 2.1标准实时计算前景色与背景色的对比度比率，提供AA/AAA合规性判定、智能推荐色值和历史记录管理功能。

- 目标用户：UI设计师、前端开发者、无障碍审核人员
- 核心价值：帮助用户快速验证配色方案的无障碍合规性，避免因对比度不足导致的可访问性问题

## 2. 核心功能

### 2.1 功能模块

1. **主面板页**：颜色输入、对比度计算结果、模拟预览、推荐色值、历史记录

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 主面板 | 颜色输入区 | 支持十六进制/rgba文本输入和原生颜色拾取器，实时验证合法性，非法值显示红色边框和错误提示 |
| 主面板 | 对比度结果区 | 显示对比度比率（保留两位小数），依据WCAG 2.1标准给出AA（4.5:1）和AAA（7:1）通过/失败判定，针对正常文本、大文本、图形组件三种规模分别判定，通过显示绿色✓徽章，失败显示红色✗徽章 |
| 主面板 | 模拟预览区 | 以当前配色渲染四种文本样式：正常正文（16px/400）、标题（24px/700）、小字（12px/300）、高亮链接，用户可自定义预览文本内容 |
| 主面板 | 推荐色值区 | 当配色不满足AA标准时，保持色相不变调整亮度生成至少3个推荐色值，圆形色块展示，点击可应用到前景色 |
| 主面板 | 历史记录侧栏 | 自动保留最近10次检测记录（前景色、背景色、对比度比率、时间戳），卡片形式展示，点击一键恢复，支持清空，数据存localStorage |

## 3. 核心流程

用户输入/选取前景色和背景色 → 实时验证颜色合法性 → 计算对比度比率 → 判定WCAG AA/AAA合规性 → 展示通过/失败徽章 → 若不合规则生成推荐色值 → 同步更新模拟预览 → 自动保存历史记录

```mermaid
    "验证颜色合法性" --> "合法" --> "计算对比度比率"
    "验证颜色合法性" --> "非法" --> "显示错误提示"
    "计算对比度比率" --> "WCAG AA/AAA判定"
    "WCAG AA/AAA判定" --> "渲染通过/失败徽章"
    "WCAG AA/AAA判定" --> "更新模拟预览"
    "WCAG AA/AAA判定" --> "不满足AA?" --> "是" --> "生成推荐色值"
    "WCAG AA/AAA判定" --> "保存历史记录"
    "生成推荐色值" --> "用户点击推荐色块"
    "用户点击推荐色块" --> "应用到前景色"
    "应用到前景色" --> "用户输入颜色"
```

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：中性浅灰（#F5F5F5）背景，深蓝（#1E3A5F）品牌色
- 按钮风格：圆角矩形，微弱阴影
- 字体：选择具有设计感的字体，正文使用清晰的无衬线体
- 布局：双列布局，左侧输入区和预览区，右侧历史记录和推荐色值
- 输入框：渐变边框动画，合法时渐变绿色，非法时渐变红色
- 徽章动画：淡入 + 轻微缩放效果
- 推荐色值卡片：悬浮放大1.05倍 + 阴影加深，点击水波纹反馈

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 主面板 | 颜色输入区 | 双列颜色输入框（文本+拾取器），渐变边框动画，错误提示文字 |
| 主面板 | 对比度结果区 | 大号对比度数字，AA/AAA徽章（绿色✓/红色✗），三级合规判定（正常文本/大文本/图形组件） |
| 主面板 | 模拟预览区 | 四种文本样式卡片，可编辑预览文本 |
| 主面板 | 推荐色值区 | 圆形色块列表，悬浮放大动画，点击水波纹 |
| 主面板 | 历史记录侧栏 | 卡片列表，清空按钮，时间戳，色块缩略图 |

### 4.3 响应式设计

- 桌面端（≥768px）：双列布局，左侧输入+结果+预览，右侧推荐+历史
- 移动端（<768px）：上下排列，顶部输入和结果，底部历史记录，所有组件保持圆润边角和统一间距

## 1. 产品概述
徒步旅行者路线规划与旅行日志应用，解决传统纸质地图无法整合个人标记、旅行后分享路线和回忆困难的问题。
- 目标用户：徒步旅行爱好者、户外探险者
- 核心价值：直观的地图路线规划、沿途记录、一键导出完整旅行数据

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 普通用户 | 无需注册 | 使用所有功能，本地存储数据 |

### 2.2 功能模块
1. **地图模块**：Canvas等高线地形图绘制、途经点添加与拖拽、路线连线显示
2. **记录模块**：途经点记录卡片、时间戳、描述文本、照片占位
3. **日志模块**：右侧边栏日志列表、记录卡片展开/收起动画、地图联动高亮
4. **导出模块**：JSON数据导出、语法高亮代码块展示

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 主页面 | 地图区域 | Canvas绘制1:50000等高线地形图，棕色等高线、蓝色河流、绿色森林，等高线间距50米 |
| 主页面 | 途经点交互 | 点击添加途经点（橙红色圆点+高度标签）、拖拽调整位置、折线实时更新 |
| 主页面 | 记录卡片 | 点击途经点弹出卡片，包含时间戳、文本框、照片按钮，最多3条记录 |
| 主页面 | 日志面板 | 右侧边栏30%宽度，时间顺序展示记录卡片，展开/收起动画0.3s |
| 主页面 | 导出功能 | 底部按钮导出JSON，代码块语法高亮（关键字蓝色、字符串绿色、数字橙色） |

## 3. 核心流程
用户打开应用 → 在地图上点击添加途经点 → 拖拽调整路线 → 点击途经点添加记录（描述+照片）→ 右侧日志实时更新 → 点击导出生成完整旅行数据

```mermaid
flowchart LR
    A["打开应用"] --> B["点击地图添加途经点"]
    B --> C["拖拽调整路线"]
    C --> D["点击途经点添加记录"]
    D --> E["输入描述+上传照片"]
    E --> F["右侧日志面板更新"]
    F --> G["点击导出按钮"]
    G --> H["生成JSON数据并展示"]
```

## 4. 用户界面设计
### 4.1 设计风格
- 主背景色：#F5F0E1（自然泥土色）
- 地图边框：#8B7355（深褐色）
- 导航栏/强调色：#2E6B2E（深绿色）
- 途经点未记录：灰色圆点
- 途经点已记录：实心绿色圆点
- 途经点标记：橙红色圆点 + 高度标签
- 路线连线：半透明红色折线
- 点击动画：白色涟漪（半径40px，0.5秒渐隐）
- 卡片悬浮：translateY -2px，box-shadow加深
- 字体：系统字体，清晰可读为主

### 4.2 页面设计概述
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 主页面 | 地图区域 | Canvas画布、等高线、河流、森林、途经点、路线折线 |
| 主页面 | 日志面板 | 垂直虚线分隔线（深褐色1px）、记录卡片列表、时间戳、缩略图、描述 |
| 主页面 | 记录卡片弹窗 | 模态框、时间戳显示、文本输入框、照片按钮、保存按钮 |
| 主页面 | 导出区域 | 导出按钮、代码块容器、语法高亮JSON |

### 4.3 响应式
- 桌面端（≥768px）：地图70%宽度居中，日志面板30%宽度右侧
- 移动端（<768px）：地图全宽，日志面板折叠为底部抽屉，汉堡菜单按钮弹出
- 触摸优化：增大点击区域，支持触摸拖拽

### 4.4 性能要求
- 地图绘制帧率稳定≥30fps
- 途经点拖拽响应延迟≤100ms
- Canvas使用requestAnimationFrame优化绘制

## 5. 技术约束
- TypeScript + React + Vite
- 依赖：react, react-dom, typescript, vite, @vitejs/plugin-react, uuid
- 不使用后端服务，所有数据本地存储
- Canvas 2D API绘制地图
- CSS transition实现动画效果

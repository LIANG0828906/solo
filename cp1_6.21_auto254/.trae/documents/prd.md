## 1. 产品概述

节奏卡牌对战游戏是一款基于浏览器的实时多人卡牌对战游戏，融合音乐节拍机制与卡牌策略玩法。玩家通过在准确把握音乐节拍出牌，与对手实时对战。

- 解决传统回合制卡牌游戏缺乏音乐节拍互动、单机卡牌难以与朋友实时竞技的问题
- 面向喜欢音乐游戏与卡牌游戏的玩家群体，提供节奏感与策略性结合的游戏体验

## 2. 核心功能

### 2.1 用户角色

| 角色 | 登录方式 | 核心权限 |
|------|----------|----------|
| 玩家 | 无需登录，直接进入匹配 | 进行实时对战、查看战报 |

### 2.2 功能模块

1. **游戏主界面**：玩家匹配、等待对手场地渲染
2. **对战界面**：上下半场玩家场地、中央节拍指示器、卡牌手牌区、出牌动画Canvas
3. **战报界面**：出牌时间轴、伤害统计、连击奖励

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|----------|----------|
| 游戏主界面 | 玩家匹配模块 | 支持两名玩家实时联机（WebSocket），每人初始手牌5张 |
| 游戏主界面 | 节拍引擎 | 节拍指示器动态闪烁，BPM=120，32小节打击乐 |
| 游戏主界面 | 卡牌系统 | 卡牌属性（攻击力、防御力）、特殊音效、节拍判断 |
| 游戏主界面 | 出牌动画 | 卡牌飞入对手场地中央，播放音效和粒子特效 |
| 战报界面 | 时间轴 | 出牌时间轴展示 |
| 战报界面 | 伤害统计 | 伤害统计和连击奖励说明 |

## 3. 核心流程

玩家进入游戏 -> WebSocket连接 -> 等待对手匹配 -> 游戏开始 -> 节拍循环（抽牌 -> 节拍窗口 -> 玩家出牌 -> 判断节拍命中判定 -> 伤害计算 -> 动画播放 -> 生命值更新 -> 游戏结束 -> 战报展示

```mermaid
flowchart LR
    A["进入游戏] --> B["WebSocket连接"]
    B --> C["等待对手匹配"]
    C --> D["游戏开始"]
    D --> E["抽牌"]
    E --> F["节拍窗口"]
    F --> G["玩家出牌"]
    G --> H["节拍命中判定"]
    H --> I["伤害计算"]
    I --> J["动画播放"]
    J --> K["生命值更新"]
    K --> L{"游戏结束？"]
    L -- 否 --> E
    L -- 是 --> M["战报展示"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深色赛博朋克风格，背景 #0F172A
- **强调色**：
  - 节拍命中：#10B981（绿色）
  - 未命中：#EF4444（红色）
  - 主渐变：#3B82F6 -> #8B5CF6
  - 光圈颜色：#F59E0B / #10B981
- **卡牌样式**：圆角12px，内边距8px
- **生命值条**：渐变条纹，宽度100%，高度16px
- **字体**：采用现代无衬线字体，标题粗体
- **布局**：固定宽高比16:9，居中显示
- **最小宽度**：1024px

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|---------|----------|--------|
| 游戏主界面 | 对手场地（上半场） | 手牌区、生命值条 |
| 游戏主界面 | 玩家场地（下半场） | 手牌区、生命值条、出牌按钮 |
| 游戏主界面 | 中央节拍指示器 | 圆形直径80px，径向渐变背景，外围旋转光圈 |
| 游戏主界面 | 卡牌 | 圆角卡牌，属性显示在底部 |
| 战报界面 | 时间轴 | 带节拍标注，命中绿色，未命中红色 |
| 战报界面 | 伤害统计 | 数值展示，连击奖励说明 |

### 4.3 响应式

- 桌面端优先设计
- 屏幕宽度小于1024px时，减小卡牌尺寸和字体
- 触摸设备适配

### 4.4 动画效果

- 节拍指示器闪烁周期0.5s
- 光圈旋转周期与BPM同步
- 生命值条平滑过渡0.3s ease-out
- 卡牌飞入动画
- 粒子特效：30-50粒子，颜色渐变，扩散半径60px，持续0.6s

## 5. 性能要求

- 实时联机延迟低于200ms（模拟）
- 卡牌动画帧率不低于45fps
- WebSocket实时通信

## 1. 产品概述

图片批量压缩转换Web应用，解决用户手动使用PS或在线工具逐张处理图片效率低下的问题。支持用户上传多张图片（表情包、贴图等），自动批量压缩并转换为WebP格式，提供在线预览和打包下载功能。

- 目标用户：需要批量处理图片的设计师、运营人员、内容创作者
- 核心价值：提升图片处理效率，一键完成批量压缩与格式转换

## 2. 核心功能

### 2.1 功能模块

1. **图片上传模块**：支持拖拽和点击上传，格式校验，数量和大小限制
2. **缩略图展示模块**：网格布局展示，显示文件名和大小
3. **压缩处理模块**：Canvas重采样压缩，质量选择，进度显示
4. **预览模块**：全尺寸图片预览弹窗
5. **导出模块**：WebP格式转换，ZIP打包下载
6. **统计面板**：实时显示处理统计数据

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|---------|---------|---------|
| 主页面 | 统计面板 | 顶部展示总图片数、总原始大小、总压缩后大小、平均压缩率 |
| 主页面 | 上传区域 | 虚线边框拖拽区，支持PNG/JPG/GIF，最多10张，单张≤2MB |
| 主页面 | 质量选择 | 下拉菜单选择：低(50%)/中(70%)/高(85%)/无损(100%) |
| 主页面 | 缩略图网格 | 120x120px缩略图，圆角8px带阴影，显示文件名和大小 |
| 主页面 | 进度条 | 整体压缩进度，每完成一张更新百分比 |
| 主页面 | 对比标签 | 缩略图右上角显示压缩前后大小，>40%绿色，否则黄色 |
| 主页面 | 预览弹窗 | 点击缩略图查看全尺寸，居中显示，最大90%窗口 |
| 主页面 | 导出功能 | 勾选图片（默认全选），导出WebP并打包ZIP下载 |

## 3. 核心流程

用户上传图片 → 选择压缩质量 → 自动开始压缩 → 查看压缩结果和统计 → 勾选需要导出的图片 → 点击导出按钮 → 自动下载ZIP包

```mermaid
    D --> E["勾选导出图片"]
    E --> F["转换为WebP格式"]
    F --> G["打包ZIP并下载"]
```

## 4. 用户界面设计

### 4.1 设计风格

- **布局风格**：卡片式布局，背景浅灰色(#f5f5f5)，卡片白色(#fff)
- **主色调**：蓝色(#1976D2)
- **辅助色**：绿色(#4CAF50) - 高压缩比，黄色(#FFC107) - 低压缩比
- **上传区**：虚线边框(#2196F3)，背景(#E3F2FD)，拖拽时实线加粗
- **毛玻璃效果**：统计面板使用backdrop-filter: blur(8px)
- **动画过渡**：cubic-bezier(0.4, 0, 0.2, 1)，时长0.3秒
- **按钮效果**：悬停背景加深10%，阴影扩大
- **进度条**：蓝色到绿色渐变

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|---------|---------|-------|
| 主页面 | 统计面板 | 毛玻璃效果，四列数据卡片，实时更新 |
| 主页面 | 上传区域 | 虚线边框，拖拽高亮，文件格式提示 |
| 主页面 | 质量选择 | 下拉菜单，默认中等质量 |
| 主页面 | 缩略图网格 | 响应式布局，悬停效果，对比标签 |
| 主页面 | 进度条 | 顶部渐变进度条，百分比显示 |
| 主页面 | 预览弹窗 | 居中显示，关闭按钮，遮罩层 |
| 主页面 | 导出按钮 | 蓝色主按钮，悬停效果 |

### 4.3 响应式设计

- **桌面端**：缩略图多列网格布局
- **手机端(<768px)**：缩略图改为2列布局，统计面板改为水平滚动卡片
- **触摸优化**：按钮和可点击区域适配触摸操作

### 4.4 性能要求

- 每张图片压缩耗时不超过2秒（以1MB图片为标准）
- 导出ZIP操作不阻塞UI

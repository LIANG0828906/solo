# 北宋铸钱工坊 - 项目架构说明

## 文件结构与职责

```
auto302/
├── package.json              # 项目依赖配置
├── vite.config.js            # Vite构建配置
├── tsconfig.json             # TypeScript编译配置
├── index.html                # 入口HTML页面
├── 项目架构说明.md            # 本文档
└── src/
    ├── main.tsx              # React应用入口
    ├── App.tsx               # 主应用组件（状态管理）
    ├── styles.css            # 全局样式
    ├── types.ts              # TypeScript类型定义
    └── components/
        ├── StepPanel.tsx     # 步骤面板组件（核心交互）
        └── CoinResult.tsx    # 结果展示组件（Canvas绘图）
```

## 数据流向图

```
用户交互
   ↓
StepPanel.tsx (子组件)
   ↓ 调用 onStepComplete(stepIndex, data)
App.tsx (父组件)
   ↓ 更新状态
   ├─→ activeStep (当前步骤)
   ├─→ completedSteps (完成标志)
   ├─→ selectedPattern (选中纹样)
   ├─→ coinData (铜钱数据)
   ├─→ castingRecord (铸造记录)
   └─→ finalImageData (最终图片)
   ↓ 向下传递props
StepPanel.tsx / CoinResult.tsx
   ↓ 渲染UI
浏览器DOM
```

## 文件调用关系

### 1. main.tsx → App.tsx
- 职责：React渲染入口
- 调用：`ReactDOM.createRoot().render(<App />)`
- 数据流：无输入，输出为挂载到#root的React树

### 2. App.tsx → StepPanel.tsx
- 传递props：
  - `activeStep: number` - 当前激活步骤索引(0-4)
  - `completedSteps: boolean[]` - 各步骤完成状态
  - `selectedPattern: CoinPattern | null` - 选中的铜钱纹样
  - `generateCoinData: () => CoinData[]` - 生成随机铜钱数据的函数
  - `onStepComplete: (stepIndex: number, data?: unknown) => void` - 步骤完成回调

### 3. App.tsx → CoinResult.tsx
- 传递props：
  - `coinData: CoinData[]` - 5枚铜钱的详细数据
  - `patternName: string` - 纹样名称
  - `castingText: string` - 铸造纪事文本
  - `finalImageData: string` - Canvas生成的图片Base64
  - `onImageGenerated: (dataUrl: string) => void` - 图片生成回调
  - `onRestart: () => void` - 重新开始回调

### 4. StepPanel.tsx 内部调用
- 步骤0（制范）：`handleMoldClick()` → `spawnParticles()` → `onStepComplete(0, patternId)`
- 步骤1（浇铸）：`handleStartCasting()` → `onStepComplete(1)`
- 步骤2（取钱）：`handleRevealCoin()` → `spawnParticles()` → `onStepComplete(2)`
- 步骤3（打磨）：`handlePolishMouseDown/Move/Up()` → `updatePolishProgress()` → `onStepComplete(3)`
- 步骤4（穿绳）：`handleStartStringing()` → `generateCoinData()` → `onStepComplete(4, coinData)`

### 5. CoinResult.tsx 内部调用
- `generateCanvasImage()` → `drawCoin()` → `generateCoinGradient()` → `onImageGenerated(dataUrl)`
- `handleDownload()` → `saveAs(finalImageData, filename)`

## 状态管理说明

| 状态名 | 类型 | 说明 | 更新时机 |

## 关键数据流细节

### 制范步骤数据流向
```
用户点击模具
   ↓
handleMoldClick(patternId)
   ├─→ 设置 selectedMold = patternId
   ├─→ 播放模具合拢动画 (0.5s)
   ├─→ 生成碰撞粒子 (12个金色粒子)
   └─→ 调用 onStepComplete(0, patternId)
         ↓
App.tsx 接收
   ├─→ 设置 completedSteps[0] = true
   ├─→ 设置 selectedPattern = patternId
   └─→ 500ms后 activeStep = 1
```

### 打磨步骤数据流向
```
用户鼠标拖拽
   ↓
handlePolishMouseMove(e)
   ├─→ 计算鼠标相对于圆心的角度
   ├─→ 将角度加入 coveredAnglesRef Set
   ├─→ 随机生成铜屑粒子
   └─→ updatePolishProgress()
         ├─→ 计算 progress = (覆盖角度数 / 36) * 100
         ├─→ 设置 polishProgress
         └─→ 若 progress >= 70，调用 onStepComplete(3)
               ↓
App.tsx 接收
   ├─→ 设置 completedSteps[3] = true
   ├─→ 记录 polishEndTime
   └─→ 500ms后 activeStep = 4
```

### 穿绳步骤数据流向
```
点击开始穿绳
   ↓
handleStartStringing()
   ├─→ 依次显示5枚铜钱 (每枚间隔300ms)
   └─→ 1800ms后
         ├─→ 调用 generateCoinData() 生成随机数据
         │    └─→ 每枚铜钱: wearLevel(0.1-0.4), patinaLevel(0.2-0.6), rotation(-5°~5°)
         └─→ 调用 onStepComplete(4, coinData)
               ↓
App.tsx 接收
   ├─→ 设置 completedSteps[4] = true
   ├─→ 设置 coinData = coinData
   ├─→ 生成 castingRecord（包含打磨时长）
   └─→ 设置 isComplete = true
```

## 性能优化点

1. **粒子动画**：使用 `requestAnimationFrame` 驱动，通过 `useRef` 存储ID避免重渲染
2. **打磨进度**：使用 `useRef` 存储 `coveredAngles`，避免频繁状态更新
3. **Canvas绘图**：仅在结果页生成一次，使用 `display: none` 预渲染
4. **步骤切换**：使用 `AnimatePresence` 实现平滑过渡，避免不必要的重挂载
5. **响应式布局**：使用CSS媒体查询，在<768px时切换为竖排布局

# 个人订阅服务管理应用 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
一款帮助用户管理个人订阅服务（流媒体、云存储、健身会员等）的Web应用，提供智能开支分析与到期提醒功能，解决用户难以追踪多个周期性订阅付费时间、金额变化以及容易忘记取消试用导致扣费的痛点。

### 1.2 目标用户
- 拥有多个订阅服务的都市白领
- 注重个人财务管理的用户
- 经常使用免费试用但忘记取消的用户

### 1.3 核心价值
- 统一管理所有订阅，一目了然
- 智能开支分析，优化消费结构
- 到期及时提醒，避免不必要扣费

---

## 2. 功能需求

### 2.1 订阅仪表盘
- **布局**：卡片网格布局展示所有订阅项
- **卡片规格**：宽度280px，圆角16px
- **卡片内容**：
  - 服务Logo（emoji或首字母占位）
  - 服务名称
  - 计费周期（月/季/年）
  - 费用金额
  - 到期日期（颜色标识：7天内#EF4444红色，30天内#F97316橙色，其余#6B7280灰色）
- **状态开关**：卡片右下角开关按钮控制订阅启用/暂停状态，切换时带有0.3秒ease-out平滑过渡动画

### 2.2 新增订阅表单
- **触发方式**：点击右下角悬浮绿色（#10B981）圆形加号按钮
- **模态窗样式**：从屏幕底部弹出，半透明高斯模糊背景（backdrop-blur-md），圆角16px
- **表单字段**：
  - 服务名称（文本输入）
  - 类别下拉框（流媒体/云存储/健身/软件/其他）
  - 计费周期选择器（月/季/年），切换时自动计算年均费用
  - 金额输入（数字，带货币符号前缀#22C55E）
  - 到期日期选择器
  - 试用提醒开关
- **提交反馈**：卡片以缩放动画（scale 0.9→1.0）出现在列表最前面

### 2.3 开支分析面板
- **入口**：导航栏"分析"标签页
- **布局**：三行布局
  - **顶部**：两张圆角12px折线图卡片（各占50%宽度）
    - 近12个月订阅总支出趋势（线条#3B82F6）
    - 各类别占比趋势（线条#8B5CF6）
    - 悬停显示Tooltip，带淡入淡出动画
  - **中间**：堆叠柱状图，展示每月各类别花费对比
  - **底部**：搜索框，输入关键词实时过滤订阅列表，0.2秒防抖延迟

### 2.4 到期提醒与通知机制
- **顶部横幅**：固定浅黄色（#FEF3C7）横幅
  - 7天内到期订阅提醒，显示数量和服务列表及倒计时
  - 点击列表项平滑滚动到对应卡片，红色虚线2px边框闪烁（0.5s频率）
  - 可手动关闭，最小化为右上角铃铛图标（#F59E0B）带红色角标
  - 点击铃铛重新展开横幅

### 2.5 数据持久化与导出
- **本地存储**：所有数据保存到LocalStorage，刷新自动恢复
- **CSV导出**：导出所有订阅信息（服务名、类别、周期、金额、到期日、状态）
- **导出反馈**：0.5秒toast提示"导出成功"，绿色打勾图标

## 3. 界面设计规范

### 3.1 配色方案
| 用途 | 颜色值 |
|------|--------|
| 主背景色 | #F8FAFC |
| 卡片背景 | #FFFFFF |
| 主文本 | #1E293B |
| 次要文本 | #64748B |
| 强调色 | #3B82F6 |
| 错误/到期色 | #EF4444 |
| 成功色 | #10B981 |
| 横幅背景 | #FEF3C7 |

### 3.2 圆角规范
- 按钮/输入框：8px
- 卡片：12px
- 模态窗：16px

### 3.3 交互反馈
- hover状态：阴影加深、轻微上浮
- active状态：缩放0.98
- focus状态：蓝色轮廓（#3B82F6）

### 3.4 响应式断点
- 平板（≥768px）：多列布局
- 手机（<768px）：单列卡片布局，侧边栏折叠为汉堡菜单

---

## 4. 性能要求
- 50个订阅卡片滚动帧率≥55fps
- 搜索过滤响应时间≤200ms

---

## 5. 技术栈
- 前端：React 18 + TypeScript + Vite
- 图表：Recharts
- 后端：Node.js + Express
- 数据存储：LocalStorage（前端）

---

## 6. 运行方式
```bash
# 前端
npm install
npm run dev

# 后端（另一个终端）
node server/index.js
```

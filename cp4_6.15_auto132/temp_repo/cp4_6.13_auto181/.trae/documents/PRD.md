# 扩散泡泡龙游戏 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
传统泡泡龙游戏采用固定关卡设计，玩家容易产生审美疲劳。本项目利用**扩散模型算法**在浏览器端实时生成独特的泡泡布局，让每一关都具有独特性和美感，提升游戏的重玩价值。

### 1.2 核心价值
- **独特性**：每关通过扩散算法随机生成，永不重复
- **美感**：通过20步扩散迭代产生自然渐变的色彩分布
- **实时生成**：生成过程动画化展示，兼具观赏性和可玩性

### 1.3 目标用户
- 休闲游戏爱好者
- 喜欢新鲜感和随机性的玩家
- 对游戏视觉效果有要求的用户

## 2. 功能需求

### 2.1 关卡生成系统
**功能描述**：使用简化的扩散过程生成独特的泡泡布局

| 需求项 | 详细说明 |
|--------|----------|
| 网格规格 | 50列 × 30行的网格系统 |
| 扩散迭代 | 20步扩散迭代，每步根据邻居颜色分布调整 |
| 重力下沉 | 每步迭代后施加重力，泡泡下落至最低可用行 |
| 颜色系统 | 5种鲜艳色：#FF6B6B、#4ECDC4、#FFE66D、#A29BFE、#FD79A8 |
| 生成动画 | 每步迭代展示在左侧预览区，每步耗时≤50ms |
| 帧率要求 | 生成动画帧率≥15FPS |

### 2.2 游戏核心玩法
**功能描述**：经典泡泡龙玩法，结合现代视觉效果

| 需求项 | 详细说明 |
|--------|----------|
| 瞄准系统 | 鼠标控制瞄准角度，显示白色半透明(alpha 0.6)虚线瞄准线 |
| 发射机制 | 点击发射泡泡，泡泡沿直线飞行 |
| 碰撞检测 | 六边形格子算法，精度0.5px |
| 消除规则 | 3个及以上同色泡泡相连即消除 |
| 掉落机制 | 消除后上方未连接的泡泡自动掉落 |
| 通关条件 | 消除所有泡泡即通关 |

### 2.3 视觉效果系统
**功能描述**：丰富的动画和粒子效果

| 需求项 | 详细说明 |
|--------|----------|
| 消除动画 | 每个泡泡分裂成8个小粒子，半径从12px缩小到0，持续0.4秒 |
| 掉落动画 | 掉落速度100px/s，重力加速度300px/s² |
| 得分动画 | 数字变化时缩放1.1倍再恢复，持续0.2秒 |
| 粒子限制 | 同时存在的粒子不超过200个，超出时优先移除最旧粒子 |

### 2.4 数据持久化
**功能描述**：玩家进度和关卡数据跨会话保存

| 需求项 | 详细说明 |
|--------|----------|
| 关卡保存 | 通关后自动保存：关卡编号、布局矩阵、得分、用时 |
| 数据查询 | 支持查询历史关卡和统计数据 |
| 持久化存储 | SQLite数据库存储，文件名为levels.db |

### 2.5 性能要求
- 游戏运行帧率稳定≥55FPS（Chrome桌面端）
- 生成动画帧率≥15FPS
- 粒子总数限制≤200个

## 3. 界面设计

### 3.1 整体布局
- **左侧**：关卡生成预览区（300px宽，白色边框#ddd，圆角12px）
- **右侧**：游戏区（剩余宽度，最小600px）
- **顶部**：信息栏（半透明黑色渐变条）

### 3.2 配色方案
| 区域 | 颜色 |
|------|------|
| 整体背景 | 深蓝色 #1a1a2e |
| 游戏区背景 | 深蓝到紫色渐变 |
| 信息栏背景 | 半透明黑色渐变 |
| 泡泡颜色 | #FF6B6B、#4ECDC4、#FFE66D、#A29BFE、#FD79A8 |
| UI边框 | #ddd |

### 3.3 交互设计
- **按钮**：圆角设计，悬停时0.2s颜色加深和微微放大，阴影变化
- **卡片**：圆角卡片设计
- **瞄准线**：白色虚线，alpha 0.6

### 3.4 信息栏内容
- 当前关卡编号
- 已发射泡泡数
- 当前得分

## 4. 技术架构概要

### 4.1 技术栈
- **前端**：React 18 + TypeScript + Vite
- **后端**：Node.js + Express
- **数据库**：SQLite (better-sqlite3)
- **构建工具**：Vite

### 4.2 核心模块
1. **关卡生成模块** (`src/generator/gridGenerator.ts`) - 扩散算法实现
2. **游戏主板模块** (`src/game/GameBoard.tsx`) - 游戏逻辑和渲染
3. **粒子系统** (`src/utils/particles.ts`) - 粒子效果管理
4. **后端API** (`server/server.ts`) - 数据持久化接口

### 4.3 API设计
- `POST /api/levels` - 保存关卡数据
- `GET /api/levels` - 获取历史关卡列表
- `GET /api/stats` - 获取游戏统计数据

# 团队代码提交活跃度热力图看板 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
小型团队需要一个研发效能看板，帮助团队成员快速查看个人本周的代码提交趋势和活跃度分布，区别于传统的项目级视图，更聚焦于个人维度的工作展示。

### 1.2 产品定位
面向研发团队的个人维度代码活跃度可视化看板，以热力图为核心展示方式，辅以详情面板和统计指标，提供直观、交互友好的提交数据查看体验。

### 1.3 目标用户
- 团队开发人员
- 技术负责人/项目经理
- Scrum Master

---

## 2. 功能需求

### 2.1 数据接口层
- 后端提供模拟数据API（GET /api/activity）
- 返回5-8名成员在最近7天的提交记录
- 每条记录包含：成员姓名、日期、提交次数、时间戳、提交信息摘要

### 2.2 热力图展示
- 纵轴：团队成员（每行对应一名成员）
- 横轴：最近7天日期（每日一格）
- 格子填充色随提交次数渐变：浅绿(#90ee90) → 橙色(#ffa500) → 深红(#8b0000)
- 悬浮显示工具提示：成员名、日期、提交次数
- 悬停高亮效果：亮度提升 + 轻微放大动画
- 点击方格触发详情面板

### 2.3 详情面板
- 从右侧滑入，展示该成员当日所有提交列表
- 内容：提交时间点、提交描述
- 动画效果：淡入 + 滑动过渡
- 毛玻璃背景效果（backdrop-filter: blur(10px)）
- 卡片圆角12px，柔和阴影

### 2.4 统计指标栏
- 本周团队总提交次数
- 人均提交次数
- 数字递增动画（从0到最终值）

### 2.5 搜索过滤
- 按成员姓名关键字搜索
- 匹配行保留，其他行淡出隐藏
- 带过渡动画效果

### 2.6 响应式布局
- ≥768px：标准热力图网格布局
- <768px：垂直滚动列表，每行显示两天数据

---

## 3. 界面设计规范

### 3.1 主题
- 深色主题，背景色：#1a1a2e
- 所有交互元素平滑过渡：transition: all 0.3s ease

### 3.2 颜色方案
| 用途 | 色值 |
|------|------|
| 主背景 | #1a1a2e |
| 低提交（浅绿）| #90ee90 |
| 中提交（橙色）| #ffa500 |
| 高提交（深红）| #8b0000 |
| 卡片/面板背景 | rgba(255,255,255,0.05) ~ rgba(255,255,255,0.1) |

### 3.3 动画规范
- 过渡时长：0.3s
- 缓动函数：ease
- 数字递增动画：requestAnimationFrame
- 热力图格子：hover时 scale(1.05) + brightness(1.2)

---

## 4. 性能约束
- 热力图初始绘制：≤300ms
- 搜索过滤响应：≤50ms
- 动画帧率：≥60fps

---

## 5. 技术栈
- **前端**：React 18 + TypeScript + Vite
- **数据可视化**：D3.js
- **HTTP客户端**：Axios
- **后端**：Node.js（模拟数据API）

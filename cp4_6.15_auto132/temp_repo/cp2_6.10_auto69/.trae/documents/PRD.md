# 周髀日晷模拟 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目背景
本项目是一个基于浏览器的古代日晷投影与时辰推算交互可视化应用。用户将扮演周代司仪的角色，在虚拟的青铜表圭（日晷）前，通过交互式操作理解中国古代的时辰推算原理。

### 1.2 项目目标
- 以3D可视化方式展示古代日晷的工作原理
- 让用户直观理解节气、太阳位置与时辰的关系
- 提供沉浸式的中国古代科技文化体验
- 实时计算并展示十二时辰及刻数

### 1.3 目标用户
- 历史文化爱好者
- 教育工作者和学生
- 对中国古代科技感兴趣的公众

## 2. 产品特性

### 2.1 核心功能

#### 2.1.1 3D日晷场景
- 圆形地平面（直径10单位），带有东南西北方位标记
- 青铜色表杆（高8单位），顶部带尖角
- 表杆与地面夹角可调节（冬至67.5度到夏至112.5度）
- 节气选择滑块（24节气）

#### 2.1.2 太阳交互系统
- 金色太阳小球，带光晕效果
- 鼠标拖拽太阳沿半圆弧轨迹从东到西滑动
- 松开后自动吸附到最近的时刻位置（步长7.5度，对应半个时辰）
- 投影线实时显示，长度和方向随太阳位置动态变化
- 投影终点用红色圆点标记

#### 2.1.3 时辰显示系统
- 左上角半透明面板显示当前时辰（如"午时三刻"）
- 扇形进度条可视化白天流逝比例
- 表盘边缘96条刻度线（12时辰 × 8刻）
- 主刻度（时辰）金色，副刻度（刻）浅灰色

#### 2.1.4 节气联动动画
- 太阳轨迹倾角随节气平滑过渡（0.5秒）
- 最高点高度角变化（冬至27度，夏至73度）
- 光照颜色变化（冬至冷色#a4b4be，夏至暖色#f5deb3）
- 阴影透明度随节气变化

#### 2.1.5 历史时辰对照
- 右侧竖向面板显示现代时刻对照
- 内置《周髀算经》简化版数据表
- 图文列表展示古代时辰与现代24小时制对照

### 2.2 交互设计

| 交互方式 | 操作对象 | 效果 |
|---------|---------|------|
| 拖拽 | 太阳 | 沿半圆弧轨迹移动，实时更新投影 |
| 拖拽 | 节气滑块 | 调整表杆夹角，太阳轨迹平滑过渡 |
| 悬停 | 表杆/太阳 | 显示操作提示标签 |
| 释放 | 太阳 | 自动吸附到最近时刻位置 |

### 2.3 视觉设计

#### 2.3.1 配色方案
- 主背景：深褐色 #2b1a0a（竹简风格）
- 地平面：浅土色 #c4a46c
- 网格线：偏绿色 #4a6a3b
- 表杆：青铜色 #8b7d6b
- 太阳：金色 #ffaa00
- 时辰主刻度：金色 #d4a017
- 副刻度：浅灰 #999999
- 投影线：深灰 #2a2a2a（半透明）
- 投影点：红色

#### 2.3.2 UI控件风格
- 圆角矩形
- CSS linear-gradient模拟木质纹理
- 字体：楷体（serif）
- 字号：12-16px
- 动效：0.2秒ease-out弹簧动画

### 2.4 性能要求
- 太阳拖拽和节气切换时帧率 ≥ 45fps
- 节气过渡动画帧率 ≥ 30fps
- 响应式交互延迟 < 100ms

## 3. 技术规范

### 3.1 技术栈
- 前端框架：React 18 + TypeScript
- 3D渲染：Three.js + @react-three/fiber + @react-three/drei
- 状态管理：Zustand
- 构建工具：Vite
- 开发语言：TypeScript（严格模式）

### 3.2 文件结构
```
├── package.json
├── vite.config.js
├── tsconfig.json
├── index.html
└── src/
    ├── main.tsx
    ├── components/
    │   └── Sundial.tsx
    └── utils/
        └── calendar.ts
```

### 3.3 核心算法
- 投影向量计算：根据表杆角度和太阳方位角
- 时辰换算：12时辰 × 8刻 = 96刻/天
- 节气太阳轨迹：根据节气调整高度角和方位角范围

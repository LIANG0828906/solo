# 桌面图标自动艺术化排列工具 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
桌面图标自动艺术化排列工具是一款创意应用，让用户能够将图标或emoji自动排列成富有韵律的艺术图案，打破传统网格布局的呆板，为桌面增添艺术气息。

### 1.2 目标用户
- 喜欢个性化桌面的用户
- 设计师和创意工作者
- 追求视觉美感的普通电脑用户

### 1.3 核心价值
- **艺术化排列**：提供波浪、螺旋、随机散点三种艺术排列模式
- **交互友好**：支持拖放上传、实时预览、点击交互
- **主题多样**：晨曦暖色、深夜冷色、霓虹撞色三种配色方案
- **导出便捷**：一键导出为PNG壁纸

---

## 2. 功能需求

### 2.1 图标管理
| 功能点 | 详细描述 | 优先级 |
|--------|----------|--------|
| 拖放上传 | 支持拖放emoji或自定义图片（png/svg）到上传区 | P0 |
| 点击上传 | 点击上传区打开文件选择器 | P0 |
| 数量限制 | 最多支持20个图标 | P0 |
| 圆形裁剪 | 每个图标显示为60x60像素圆形裁剪，带2px浅灰色边框和圆角 | P0 |
| 缩略图列表 | 已上传图标显示为缩略图列表 | P0 |

### 2.2 排列模式
| 功能点 | 详细描述 | 优先级 |
|--------|----------|--------|
| 波浪模式 | 图标沿正弦波路径排列，相邻图标旋转角度在-15°到+15°随机变化 | P0 |
| 螺旋模式 | 从画布中心向外旋转排列，图标缩放从1.0线性递减到0.6 | P0 |
| 随机散点模式 | 画布上随机分布，间距至少20px不重叠，至少覆盖画布60%面积 | P0 |
| 实时预览 | 切换模式时画布立即重新排列 | P0 |

### 2.3 交互功能
| 功能点 | 详细描述 | 优先级 |
|--------|----------|--------|
| 图标点击 | 点击图标以0.3秒ease-out动画弹起并放大1.3倍，右上角弹出序号和名称 | P0 |
| 取消选中 | 点击其他位置或再次点击该图标取消选中 | P0 |
| 鼠标悬停 | 鼠标悬停时图标放大1.2倍 | P0 |
| 密度调节 | 滑块（50-200px）调节图标排列密度，0.5秒过渡动画 | P0 |
| 大小调节 | 滑块调节图标大小 | P0 |

### 2.4 主题系统
| 功能点 | 详细描述 | 优先级 |
|--------|----------|--------|
| 晨曦暖色 | 背景#FFF8E7，图标主色#FF6B35、#FF8C42 | P0 |
| 深夜冷色 | 背景#1A1A2E，图标主色#00D2FF、#7B2FFF | P0 |
| 霓虹撞色 | 背景#0D0D0D，图标主色#FF007F、#00FF41 | P0 |
| 平滑过渡 | 背景1秒渐变过渡，图标颜色1秒后跟随变化 | P0 |

### 2.5 导出功能
| 功能点 | 详细描述 | 优先级 |
|--------|----------|--------|
| PNG导出 | 导出当前画布为PNG壁纸 | P0 |

## 3. 界面需求

### 3.1 布局结构
- **画布区域**：左侧75%宽度，背景色随主题变化，2px深灰色边框
- **控制面板**：右侧固定300px宽度，半透明灰色背景（rgba(30,30,30,0.85)），白色文字标签
- **响应式**：宽度小于1024px时，控制面板变为可折叠抽屉，点击齿轮图标展开

### 3.2 控件样式
- **上传区**：虚线边框，拖入时变为实线并绿色高亮
- **滑块**：自定义样式，拖动时荧光色进度条（随主题变化）
- **下拉菜单**：磨砂玻璃效果（backdrop-filter: blur(10px)），悬停平滑背景色过渡
- **整体页面**：水平居中

---

## 4. 性能需求

| 指标 | 要求 |
|------|------|
| 帧率 | 20个图标动画时稳定在55FPS以上 |
| 响应时间 | 切换模式或调整密度到完成排列不超过300ms |
| 过渡动画 | 图标位置移动过渡0.5秒，背景过渡1秒 |

---

## 5. 技术栈

- **框架**：React 18 + TypeScript
- **构建工具**：Vite
- **图形绘制**：p5.js + react-p5-wrapper
- **状态管理**：React useState/useReducer

---

## 6. 数据流与模块关系

### 6.1 数据流向
```
用户操作 → ControlPanel → onChange回调 → App状态更新 → CanvasRenderer重绘
```

### 6.2 模块调用关系
```
App.tsx
├── ControlPanel.tsx (接收用户输入，回调更新状态)
│   └── 图标池、排列模式、颜色主题、密度/大小滑块
└── CanvasRenderer.tsx (接收状态，调用排列算法，绘制画布)
    └── patterns.ts (纯函数排列算法)
        ├── generateWavePattern
        ├── generateSpiralPattern
        └── generateRandomPattern
```

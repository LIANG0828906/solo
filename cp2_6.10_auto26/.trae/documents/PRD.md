## 1. 产品概述

本应用是一个基于浏览器的宋代雕版印刷工艺模拟Web应用，通过交互式方式让用户体验古代刻版匠人的工作流程——从选择手写文稿、镜像反刻梨木版、调节墨量压力进行拓印，最终生成可翻阅的书页。

- 主要目的：通过交互探索活字印刷与木版雕刻之间的关系，让用户沉浸式体验古代印刷工艺
- 目标用户：文化爱好者、学生、对传统工艺感兴趣的普通网民
- 产品价值：将非物质文化遗产数字化，以寓教于乐的方式传播宋代印刷文明

## 2. 核心特性

### 2.1 功能模块

1. **刻坊主场景**：梨木版区域、拓印区、案几布局、进度显示
2. **文稿管理**：三种笔体（楷书、行书、草书）预置文稿，拖拽缩放旋转
3. **反刻引擎**：镜像反转算法、雕刻刀路径动画、凹槽效果模拟
4. **拓印控制**：墨量滑块、压力滑块、墨迹纤维扩散模拟
5. **书页系统**：拓印书页生成、卷边效果、翻页动画、历史记录

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|-----------|-------------|---------------------|
| 主页面 | 刻坊正视图 | 青砖地面、米色墙面、梨木版放置区、拓印区、案几雕花 |
| 主页面 | 文稿操作面板 | 笔体切换按钮、文稿预览、拖拽缩放旋转控制 |
| 主页面 | 控件面板 | 垂直墨量滑块、垂直压力滑块、反刻按钮、拓印按钮 |
| 主页面 | 进度环组件 | 圆形进度环、刻版完成度百分比、小图预览 |
| 主页面 | 书页组件 | 卷边效果、拖拽翻页、历史记录（最多5张） |

## 3. 核心流程

```mermaid
flowchart LR
    A["选择文稿笔体"] --> B["拖拽调整文稿位置"]
    B --> C["点击反刻按钮"]
    C --> D["逐笔动画镜像雕刻"]
    D --> E["调节墨量与压力"]
    E --> F["实时预览拓印效果"]
    F --> G["点击拓印按钮生成书页"]
    G --> H["拖拽翻阅历史书页"]
```

**用户流程描述**：用户进入刻坊场景，从案头选取一幅宋代诗词文稿（可切换楷书/行书/草书），通过拖拽调整文稿在梨木版上的位置和角度，点击"反刻"后观察笔迹逐笔镜像雕刻到木板上，形成凹槽效果。然后调节刷墨滚轮的墨量和拓印压板的压力，实时观察墨迹在纸张纤维中的扩散效果，满意后点击"拓印"生成一张完整书页。用户可拖拽书页边缘翻阅之前的拓印作品（最多保存5张历史记录）。

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：#8b4513（木色），辅色：#f5deb3（古纸色），强调色：#3a1a0a（墨色），警告色：#cc0000（朱砂色）
- **按钮风格**：木质纹理背景，按下时box-shadow内缩2px，背景色加深10%，圆角6px
- **字体**：Georgia, serif 衬线风格，模拟古籍印刷感
- **布局风格**：上下两栏布局，上栏为刻坊正视图，下栏为操作面板
- **视觉细节**：木纹用重复线性渐变模拟（间距3px），纸张底纹用noise纹理，线段描边使用stroke-linecap: round和stroke-linejoin: round模拟毛笔质感

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI元素 |
|-----------|-------------|-------------|
| 主页面 | 刻坊场景 | 青砖地面#6b7b6b、浅米色墙面#f0e6d3、梨木版#d2b48c、案几#5d3a1a、案腿卷草纹伪元素 |
| 主页面 | 操作面板 | 深木色#4a2a1a背景、榫卯分隔线0.5px实线#8b7355、滑块tooltip动画 |
| 主页面 | 进度环 | 环宽8px、渐变#a0522d到#8b4513、25%缩略预览、悬停放大 |
| 主页面 | 书页 | border-radius卷边15度、box-shadow阴影、framer-motion翻页动画300ms |

### 4.3 响应式设计

- **桌面优先**：上下两栏布局
- **移动端（<768px）**：垂直堆叠布局，上栏55vh，下栏45vh
- **触摸优化**：滑块宽度自适应80%容器，按钮触摸区域48px×48px
- **过渡动画**：面板切换水平滑动（margin-left从-100%到0，300ms ease-in-out），部件聚焦微弱发光（0 0 6px #d2691e）

### 4.4 性能指标

- 交互响应时间 < 200ms（切换文稿、调节滑块、翻页）
- 拓印重绘帧率 ≥ 45fps
- 墨迹扩散像素遍历使用requestAnimationFrame分批执行，每次最多处理5000像素

# 促销活动实时追踪平台 - 产品需求文档

## 1. 需求概述

### 1.1 项目背景
小型电商团队在运营过程中经常需要同时运行多个促销活动，但缺乏直观的工具来实时对比各活动效果（曝光、点击、转化、ROI），导致难以及时调整活动策略。

### 1.2 目标用户
小型电商运营团队、市场营销人员

### 1.3 核心价值
- 实时追踪多促销活动的关键指标
- 直观对比不同活动的表现
- 快速识别高/低效活动，及时调整策略

---

## 2. 功能需求

### 2.1 活动创建功能
| 需求项 | 描述 |
|--------|------|
| 活动名称 | 用户可输入活动名称 |
| 活动类型 | 支持三种类型：满减、折扣、秒杀 |
| 预算上限 | 设置活动的最大预算金额 |
| 时间设置 | 设定活动的开始和结束时间 |
| 成功跳转 | 创建成功后自动进入活动详情面板 |

### 2.2 概览仪表盘
| 需求项 | 描述 |
|--------|------|
| 卡片网格布局 | 以卡片形式展示所有进行中的活动 |
| 卡片样式 | 宽度280px，白色背景，圆角16px，阴影0 4px 12px rgba(0,0,0,0.06) |
| 活动信息 | 卡片顶部显示活动名称和类型标签（不同类型不同颜色） |
| 关键指标 | 显示当前曝光量、点击率(%)、实时ROI |
| 预算进度条 | 高度6px，显示预算使用百分比，80%以上橙色闪烁 |
| 筛选功能 | 按活动类型或状态（进行中/已结束/未开始）筛选 |
| 筛选动画 | 移除卡片淡出(0.2s)，新卡片上滑进入(0.3s) |

### 2.3 活动详情页
| 需求项 | 描述 |
|--------|------|
| 实时折线图 | 每小时曝光量、点击量、转化率三条平滑曲线 |
| 折线颜色 | 曝光#60A5FA，点击#34D399，转化#F472B6 |
| 图表交互 | 圆点标记，悬停显示具体数值 |
| 用户热力图 | Canvas绘制点击密集区域，色阶从#FDE68A到#EF4444 |
| 暂停/恢复 | 切换按钮控制实时更新，颜色平滑过渡0.3s |

### 2.4 视觉设计规范
| 类别 | 规范 |
|------|------|
| 主色调 | 白色背景 + 深灰文字#1F2937 |
| 辅色调 | 靛蓝#6366F1（按钮、进度条） |
| 状态色 | 成功#10B981，错误#EF4444 |
| 按钮交互 | hover时上移2px + 阴影加深（0.2s过渡） |
| 圆角 | 卡片16px，其他元素12-16px |
| 响应式 | ≥1200px三列，768-1200px两列，<768px单列 |

### 2.5 性能要求
| 要求 | 指标 |
|------|------|
| 数据轮询 | 5秒间隔 |
| 动画帧率 | ≥60fps |
| 布局稳定性 | 数据刷新无明显抖动 |

## 3. 非功能需求

### 3.1 可用性
- 界面简洁直观，操作流程清晰
- 所有数据实时可见，无需手动刷新
- 支持移动端和桌面端访问

### 3.2 可维护性
- 模块化架构，代码组织清晰
- TypeScript严格类型检查
- 样式使用CSS变量便于主题扩展

---

## 4. 用户交互流程

### 4.1 主流程
```
用户访问 → 概览仪表盘（展示所有活动卡片）
        → 点击卡片 → 进入活动详情页
        → 点击创建活动 → 填写表单 → 提交 → 进入详情页
```

### 4.2 筛选流程
```
用户选择筛选条件 → 不符合卡片淡出移除（0.2s）
                → 符合条件卡片向上滑入（0.3s）
```

### 4.3 详情页交互
```
进入详情页 → 实时数据面板（左侧） + 热力图（右侧）
          → 点击暂停/恢复按钮 → 控制实时更新
          → 悬停折线图 → 显示具体数值
```

## 5. 数据模型

### 5.1 活动(Activity)
```typescript
interface Activity {
  id: string;
  name: string;
  type: 'full_reduction' | 'discount' | 'flash_sale';
  budgetLimit: number;
  budgetUsed: number;
  startTime: Date;
  endTime: Date;
  status: 'upcoming' | 'ongoing' | 'ended';
  impressions: number;
  clicks: number;
  conversions: number;
  revenue: number;
}
```

### 5.2 小时数据(HourlyData)
```typescript
interface HourlyData {
  timestamp: number;
  impressions: number;
  clicks: number;
  conversions: number;
}
```

### 5.3 行为数据(BehaviorData)
```typescript
interface ClickPoint {
  x: number;
  y: number;
  count: number;
}
```

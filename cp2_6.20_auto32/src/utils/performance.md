# 性能优化方案

## 1. 语法检查响应时间优化（≤ 2秒）

### 1.1 前端防抖策略
- 位置：`src/utils/debounce.ts`
- 应用：`src/components/ErrorHighlighter.tsx` 第 55-59 行
- 策略：输入去抖 300ms，避免每次击键都触发请求
- 效果：减少 70%+ 的预检请求次数

### 1.2 后端规则匹配优化
- 位置：`backend/services/grammar_checker.py`
- 策略：
  - 单次扫描文本，多规则并行匹配
  - 拼写检查使用 `str.find()` 线性扫描（O(n*m)，m为规则数）
  - 标点/语法检查使用正则表达式预编译
  - 预检接口仅检查拼写+标点（约减少 30% 计算量）

### 1.3 请求优化
- 预检请求仅返回轻量数据（错误列表，无结构/评分）
- 使用 HTTP 连接复用（axios keep-alive）

## 2. 批量提交不卡顿（≤ 5篇并发）

### 2.1 前端状态管理
- Zustand 轻量状态，避免频繁重渲染
- 提交状态独立管理，不阻塞编辑区

### 2.2 并发控制
- API 层可配置并发上限（建议：5）
- 使用 `Promise.all` 批量提交时，配合后端限流

### 2.3 渲染优化
- 错误列表虚拟滚动（大数据量时）
- 图表使用 CSS `will-change: transform` 提示浏览器优化

## 3. 雷达图动画帧率（≥ 30FPS）

### 3.1 实现方式
- 位置：`src/components/ScoreRadar.tsx`
- 使用 `requestAnimationFrame` 手动驱动动画
- 每次帧更新数据，触发重绘
- 缓动函数：`easeOutCubic`（1 - (1-t)³）

### 3.2 性能保障
- 动画时长：800ms
- 理论帧率：60FPS（RAF 原生频率）
- 实际保障：≥ 30FPS（单图仅 4 个数据点，计算量极小）
- GPU 加速：`transform` + `opacity` 属性动画

## 4. 其他优化

### 4.1 首屏加载
- 路由懒加载（可选扩展）
- 字体预连接（preconnect）
- 关键 CSS 内联

### 4.2 内存管理
- 编辑器实例在卸载时自动销毁（Tiptap 内置）
- 历史记录上限：20 条（防止无限增长）

## 1. 产品概述

节奏星际（Rhythm Star）是一款通过声音节奏驱动太空飞船躲避陨石的跑酷游戏，解决传统跑酷游戏玩法单一、缺乏声画同步沉浸体验的问题。玩家通过环境音或音乐节拍控制飞船跳跃与闪避，实现音游与跑酷的深度融合。

- 目标用户：喜欢音乐游戏和跑酷游戏的休闲玩家
- 核心价值：实时音频驱动游戏玩法，创造独特的声画同步沉浸体验

## 2. 核心功能

### 2.1 功能模块

1. **游戏主界面**：全屏Canvas游戏画面、星空背景、飞船与陨石渲染、得分与BPM显示
2. **结算界面**：最终得分、存活时间、最高BPM、再来一次/分享成绩按钮

### 2.2 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 游戏主界面 | 音频采集模块 | 麦克风实时采集音频，频谱与节拍分析（延迟<50ms），输出低频/高频特征与BPM |
| 游戏主界面 | 节奏映射模块 | 低频重音→飞船向上跳跃（80px，0.4s），高频段落→水平闪避（60px，0.2s），BPM→障碍物速度 |
| 游戏主界面 | 陨石生成模块 | 三种陨石（普通/燃烧/分裂），难度递增生成规则（1.5-3s→0.8s下限） |
| 游戏主界面 | 碰撞检测模块 | 飞船与陨石碰撞检测，扣血25点，归零游戏结束 |
| 游戏主界面 | 粒子特效模块 | 飞船尾焰粒子、陨石碰撞爆炸、燃烧陨石拖尾 |
| 游戏主界面 | 背景系统 | 80颗闪烁星星、背景色随得分过渡（深蓝→暗紫→深红） |
| 游戏主界面 | 手动触发 | 点击屏幕左右半区触发30px位移，1.5s冷却 |
| 结算界面 | 数据展示 | 最终得分、存活时间、最高BPM三项数据 |
| 结算界面 | 操作按钮 | 再来一次（重置游戏）、分享成绩 |

## 3. 核心流程

玩家进入游戏→授权麦克风→音频引擎开始采集→分析频谱与节拍→映射飞船动作→陨石从右侧生成→飞船根据节奏自动闪避→玩家可手动点击辅助→碰撞扣血→血量归零→显示结算界面→再来一次重置

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深蓝#0D1B2A（星空背景），暗紫#1A0A2E，深红#2D0A0A（背景过渡）
- 强调色：黄色#FFD54F（BPM显示），橙色#FF6F00-#FFB300（尾焰粒子），红色#E53935（按钮/扣血）
- 飞船色：深蓝#1A237E，白色尾焰粒子
- 字体：'Courier New'（得分数据），等宽风格科技感
- 布局：全屏Canvas自适应，右上角HUD叠加，中央结算弹窗

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 游戏主界面 | 星空背景 | Canvas全屏，80颗闪烁星星（白色圆点0.5-2px，透明度0.2-1闪烁） |
| 游戏主界面 | 飞船 | 三角形20px边长，#1A237E，白色尾焰6-10粒子（#FF6F00→#FFB300，3-6px，0.6s生命周期），绿色生命条80x6px圆角2px |
| 游戏主界面 | 陨石 | 普通：棕色圆形16px锯齿边缘；燃烧：橙色椭圆24x16拖尾粒子；分裂：红色菱形20x20 |
| 游戏主界面 | HUD | 右上角得分（白色18px Courier New）、BPM（#FFD54F）、碰撞爆炸8条放射线橙红24px 0.3s |
| 结算界面 | 数据区 | 中央白色24px，得分/时间/BPM间距12px |
| 结算界面 | 按钮区 | 红色160x44px圆角6px #E53935 "再来一次"，灰色#616161 "分享成绩"，悬停加深+上浮2px |

### 4.3 响应式设计

- 桌面优先，Canvas全屏自适应
- 点击区域：左半屏向左闪避，右半屏向右闪避
- 所有UI元素0.2s平滑出现过渡（opacity+transform）

### 4.4 性能指标

- 60FPS稳定运行
- 音频采集到动作响应总延迟≤100ms
- Canvas渲染帧间隔≤16ms

# NutriLog 家庭饮食营养分析应用 - 产品需求文档

## 1. 产品概述

### 1.1 产品定位
NutriLog 是一款面向家庭用户的日常饮食营养分析与智能建议工具，帮助用户科学记录每餐饮食，自动计算营养成分摄入，并根据个人设定目标提供实时反馈与调整建议。

### 1.2 目标用户
- 关注健康饮食的家庭用户
- 需要控制营养摄入的健身人群
- 有慢性病饮食管理需求的用户
- 希望培养健康饮食习惯的普通用户

### 1.3 核心价值
- **便捷记录**：智能食物搜索与份量录入，30秒完成一餐记录
- **数据可视化**：实时仪表盘展示营养摄入进度，一目了然
- **智能建议**：基于目标的AI建议，帮助用户优化饮食结构
- **历史追踪**：7天饮食回顾，发现饮食习惯问题

| 需求编号 | 需求描述 | 优先级 |
|---------|---------|--------|
| FR-001 | 用户通过搜索框输入食物名称，下拉列表即时显示匹配食物 | P0 |
| FR-002 | 搜索结果展示食物每100克营养数据（热量、蛋白质、脂肪、碳水、钠） | P0 |
| FR-003 | 用户选择食物后输入份量（克），系统自动计算该份量的营养值 | P0 |
| FR-004 | 用户可选择餐次类型：早餐、午餐、晚餐、加餐 | P0 |
| FR-005 | 点击添加按钮将食物记录记入当前餐次 | P0 |

| FR-102 | 热量进度条（渐变填充）显示完成百分比 | P0 |

## 3. 非功能需求

### 3.1 性能要求
| 编号 | 指标 | 阈值 |
|-----|------|------|
| NFR-001 | 营养数据计算响应时间 | ≤ 200ms |
| NFR-002 | 仪表盘刷新响应时间 | ≤ 200ms |
| NFR-003 | 环形图/进度条动画帧率 | 60fps |
| NFR-004 | 历史记录虚拟化渲染 | 仅渲染可视区域 |
| NFR-005 | 页面首屏加载时间 | ≤ 3s |

### 3.2 UI/UX 要求
- **设计语言**：极简毛玻璃风格
- **主背景**：浅色渐变（#F0F4F8 → #D9E2EC）
- **卡片圆角**：12px
- **阴影**：统一层级 0 2px 8px rgba(0,0,0,0.06)
- **过渡动画**：统一 300ms ease-in-out
- **导航栏**：高度60px，左侧应用名，右侧设置图标
- **布局**：桌面端左右两栏，移动端响应式折叠

### 3.3 响应式设计
- **≥1024px**：左右两栏布局，左侧记录区，右侧固定仪表盘（350px）
- **768px-1024px**：仪表盘宽度自适应
- **<768px**：仪表盘折叠到底部可折叠面板

### 3.4 技术约束
- 前端：React 18 + TypeScript + Vite
- 后端：Node.js Express RESTful API
- 图表：Recharts
- 状态管理：React Hooks (useState, useContext, useReducer)

## 4. 数据模型

```typescript
interface Food {

  calories: number;      // 千卡/100g

  fat: number;           // 克/100g
  carbs: number;         // 克/100g
  sodium: number;        // 毫克/100g
}
```

### 4.2 用餐记录
```typescript
interface MealRecord {
  id: string;
  date: string;          // YYYY-MM-DD
  mealType: 'breakfast' | 'lunch' | 'dinner' | 'snack';
  foods: FoodItem[];
  createdAt: number;
}

interface FoodItem {
  foodId: string;
  name: string;
  grams: number;
  nutrition: Nutrition;
}

interface Nutrition {
  calories: number;
  protein: number;
  fat: number;
  carbs: number;
  sodium: number;
}
```

## 5. API 接口规范

| 接口 | 方法 | 描述 |
|-----|------|------|
| /api/foods/search?q=xxx | GET | 搜索食物，返回匹配列表 |
| /api/meals | POST | 添加用餐记录 |
| /api/meals?date=xxx | GET | 获取指定日期的用餐记录 |
| /api/meals/history?days=7 | GET | 获取近N天历史记录 |
| /api/goals | GET | 获取用户每日目标 |
| /api/goals | PUT | 更新用户每日目标 |
| /api/daily-summary?date=xxx | GET | 获取指定日期营养汇总 |
| /api/suggestions?date=xxx | GET | 获取智能建议 |

## 6. 交付范围

### 6.1 必须交付
- ✅ 完整前端代码（React + TypeScript）
- ✅ 后端Express服务代码
- ✅ 模拟营养数据库（至少20种常见食物）
- ✅ 所有核心功能模块
- ✅ 配置文件（package.json, tsconfig.json, vite.config.js等）
- ✅ 可通过 npm install && npm run dev 启动运行

### 6.2 不包含
- ❌ 用户登录/注册系统（单用户模式）
- ❌ 真实营养数据库接入（使用模拟数据）
- ❌ 移动端原生App（仅响应式Web）
- ❌ 数据持久化存储（使用内存存储）

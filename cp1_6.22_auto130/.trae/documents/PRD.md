# 沉浸式UI组件沙盒与交互状态调试应用 - PRD

## 1. 产品概述

### 1.1 产品背景
前端开发者在重构大型Sass平台UI组件库时，面临以下痛点：
- 手动编写mock数据和状态预览耗时耗力
- 不同组件之间的交互逻辑难以独立验证
- 缺少可视化的组件props和状态调试工具
- 组件交互过程难以复现和调试

### 1.2 产品目标
构建一个沉浸式UI组件沙盒环境，让开发者能够：
- 在独立预览环境中加载、排列多个UI组件
- 通过可视化面板实时修改组件props、样式变量和内部状态
- 录制组件交互的时序日志用于调试
- 支持撤销/重做所有修改操作

### 1.3 目标用户
前端开发者、UI组件库维护者、交互设计师

## 2. 功能需求

### 2.1 沙盒区域（左侧70%）
- 默认显示预设组件示例集：输入框、按钮、开关、表格
- 组件支持鼠标点击选中，选中后显示2px蓝色虚线边框
- 支持拖拽调整组件展示顺序（长按0.3秒触发拖拽）
- 拖拽时其他组件自动让位，平滑移动动画
- 组件之间垂直排列，间距16px

### 2.2 控制面板（右侧30%）
**顶部 - 组件属性标签页**
- 显示选中组件的所有可编辑props
- 每个prop用一行label和输入框/下拉选择器表示
- 修改后立即应用，触发组件重渲染
- 修改过程有0.2s的value变化闪烁动画

**中间 - Mock数据源标签页**
- 可增删行的表格，每行有key和value两个输入框
- value支持基本类型和JSON字符串
- 修改后同步更新到组件的数据依赖

**底部 - 事件日志面板**
- 时间倒序排列最近50条交互事件
- 每条显示事件名、时间戳和payload概览
- 点击条目展开详情

### 2.3 标签切换
- 标签按钮：组件属性 | Mock数据源 | 事件日志
- 选中状态：深蓝灰色背景（#334155），白色文字
- 未选中状态：浅灰背景（#F1F5F9），深色文字（#1E293B）
- 切换时0.3s水平滑动过渡动画

### 2.4 撤销/重做
- 所有修改通过zustand store集中管理
- 每次修改自动生成快照存入undo/redo栈
- 快捷键：Ctrl+Z撤销，Ctrl+Shift+Z重做
- 恢复过程有0.15s淡入效果

## 3. 界面设计规范

### 3.1 配色方案
- 页面背景：浅灰白色（#F8FAFC）
- 字体颜色：深灰色（#1E293B）
- 沙盒区域背景：白色（#FFFFFF）
- 沙盒边框：#E2E8F0，2px圆角
- 主色调蓝色：#2563EB（按钮背景）、#3B82F6（选中边框、焦点）、#E0F2FE（闪烁动画背景）
- 标签选中：#334155（深蓝灰）
- 标签未选中：#F1F5F9（浅灰）、#1E293B（文字）

### 3.2 布局规范
- 宽屏（≥1200px）：沙盒70%，控制面板360px
- 平板（768-1200px）：沙盒60%，控制面板300px
- 所有输入框和按钮圆角8px
- 输入框焦点：蓝色边框（#3B82F6），0.2s box-shadow过渡
- 自定义滚动条：8px宽，轨道#E2E8F0，滑块#475569

### 3.3 交互规范
- prop修改闪烁动画：白色→#E0F2FE→白色，0.2s
- 标签切换：水平滑动，0.3s
- 拖拽让位：平滑移动，0.2s
- 撤销/重做淡入：0.15s

---

## 4. 性能约束

- 沙盒渲染帧率≥45FPS（6个复杂组件时）
- prop修改到渲染更新延迟≤100ms
- 事件日志操作不阻塞UI主线程
- 日志存储上限500条，超出时丢弃最早10%

---

## 5. 技术架构概要

### 5.1 核心技术栈
- React 18 + TypeScript
- Vite 5（构建工具）
- Zustand（状态管理）
- Immer（不可变状态更新）
- UUID（唯一标识）

### 5.2 模块划分
- **模块A**：沙盒渲染与状态管理
  - SandboxRenderer.tsx - 沙盒主渲染器
  - sandboxStore.ts - Zustand状态管理
  - 内置UI组件（按钮、输入框、开关、表格）

- **模块B**：数据模拟层
  - MockDataEditor.tsx - mock数据编辑面板
  - mockStore.ts - mock数据状态管理

- **模块C**：日志记录与分析
  - InteractionLogger.ts - 日志采集模块
  - loggerStore.ts - 日志状态管理
  - EventLogPanel.tsx - 事件日志面板
